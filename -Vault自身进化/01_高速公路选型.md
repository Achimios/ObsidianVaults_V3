# 高速公路选型 — V3 导航策略

> 老维（Victor Copilot）| 2026-03-27  
> 这是 V3 建设的第一块砖，在建目录骨架前先定导航哲学。

---

## 一句话结论

**不选一条路，按场景分流：**

| 场景 | 用哪条路 | 工具 |
|---|---|---|
| 我知道文件名/路径 | 直接路径 | VS Code `Ctrl+P` / file_search |
| 我知道关键词/代码符号 | Grep 搜索 | grep_search / VS Code `Ctrl+Shift+F` |
| 我知道文件之间的关系 | WikiLink 虫洞 | Obsidian Graph / backlinks |
| 我不知道关键词，只知道概念 | 语义搜索 | semantic_search / search_subagent |
| AI 进入任务前的快速定位 | 分层 CONTEXT.md | 直接读，不搜索 |

---

## 五条高速公路详解

### 🛣️ 1. 直接路径（最快）
**适用**：你/AI 知道精确文件名  
**工具**：`Ctrl+P`、`file_search`、`obsidian files folder="X"`  
**规则**：文件名要有意义（`01_高速公路选型.md` > `notes.md`）  
**V3 约定**：重要文件加编号前缀，方便记忆和 Tab 补全

---

### 🔍 2. Grep 搜索（精准关键词）
**适用**：找函数名、配置项、出现过的词  
**工具**：`grep_search`、VS Code `Ctrl+Shift+F`、`obsidian search:context query="X"`  
**规则**：比 WikiLink 快，比语义搜索精准  
**限制**：要知道关键词才能用

---

### 🕳️ 3. WikiLink 虫洞（知识网络）
**适用**：在笔记之间跳转，建立知识图谱，反向链接  
**工具**：`[[文件名]]`、Obsidian Graph View、`obsidian backlinks file="X"`  
**规则**：链接是手动的，精准但需要维护  
**V3 约定**：只在真正有关联的地方加 link，不滥用

---

### 🧠 4. 语义搜索（不知道关键词）
**适用**：概念探索、"我想找那个关于 X 的东西"  
**工具**：`semantic_search`、`search_subagent`  
**规则**：慢但能找到模糊目标  
**成本**：占用大量 context，用完要压缩

---

### 🗺️ 5. 分层 CONTEXT.md（AI 快速上手）
**适用**：AI 进入新任务时的定向  
**机制**：每层目录一个 `CONTEXT.md`，AI 进来先读这个，不搜索  

```
V3/
├── CONTEXT.md              ← 全局：王国架构 + 各目录一句话
├── _Vault自身进化/
│   └── CONTEXT.md          ← 当前建设状态 + 下一步精确到文件
├── ~Vault空间/
│   └── CONTEXT.md          ← AI义体、项目模块导航
└── Projects/<项目>/
    └── CONTEXT.md          ← 该项目当前状态（任务开始时更新）
```

**这是斯坦福方法的核心：不搜索，直接读；读完即上手。**

---

## V3 导航规则（固定下来）

1. **AI 进入任务** → 先读 `CONTEXT.md`（根）→ 再读对应子目录 `CONTEXT.md`
2. **找代码/配置** → grep_search
3. **找关联笔记** → WikiLink / backlinks
4. **不知道找什么** → search_subagent（结果精简后再用）
5. **每次大任务开始** → 更新对应的 `CONTEXT.md`（作战地图）
6. **每次大任务结束** → 压缩 session，更新 `CONTEXT.md` 为新状态
