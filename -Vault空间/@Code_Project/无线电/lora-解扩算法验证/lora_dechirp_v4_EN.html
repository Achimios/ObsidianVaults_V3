📡 LoRa De-chirp Parameters v4

—

Signal Parameters (BW / Fc adjustable)

BW = 120 Hz, T = 0.2 s (fixed)
Fc = 250 Hz (teaching carrier, real LoRa at GHz)
Passband = 40 ~ 160 Hz
Baseband = −60 ~ +60 Hz
N = BW·T = 24 chips, Δf = 1/T = 5 Hz (fixed!)

Current State

fsymbol = s × 5 Hz = 0 Hz
Sample count Nsamp = 40
Baseband Nyquist = Fs/2 = 100 Hz
Signal max baseband freq = ±60 Hz

Why symbols 0 ~ N−1?

N = BW × T = 120 × 0.2 = 24 chips
Each symbol = cyclic shift 0~N-1; 24 possible values
Real LoRa SF7: BW=125kHz, T=1.024ms → N=128
More symbols = more bits/symbol, but demodulation costs more CPU

Aliasing Explained

Baseband signal: instantaneous freq range −60 ~ +60 Hz
Fs=400Hz: Nyquist=200Hz > 60Hz → No aliasing
Fs=200Hz: Nyquist=100Hz > 60Hz → Also no aliasing
Fs=100Hz: Nyquist=50Hz < 60Hz → Aliased!
Continuous lines are the theoretical formula (no folding shown); stems are actual ADC-sampled values (already aliased)

⚠️ Why must FFT use N_SYM chip samples?

Real LoRa: Fs = BW = chip rate, N_samp = N_SYM = 24
The two segments of the cyclic-shifted chirp (s·Δf and (s-N)·Δf) in a 24-pt DFT
align perfectly because e^{-j2πn}=1, so they add up to the same bin s
Using Nsamp=40 (Fs=200Hz) misaligns the two segments on the DFT grid
→ Spectral leakage! False correlation peaks on both sides
⑨ FFT plot always uses N_SYM=24 chip-rate samples

RF-side "true" waveform Re{rBB(t)·ej2πFct}, sweep 40→160 Hz. This is an analog signal, DSP never sees it.
③ Received r(t): Baseband I/Q ★ (what DSP actually sees)
Hardware IQ downconversion + ADC yields complex sequence r[n] = I[n] + jQ[n]. Dots = ADC samples (with noise). When aliasing: dashed=true continuous signal, colored solid=clean sinc reconstruction (shows aliasing shape). SNR<∞: gray solid = noisy sinc reconstruction, shows how noise alters the waveform (sinc weighting averages and partially smooths noise).

④ Conjugate down-chirp c*(t): Passband Reference Waveform
Instantaneous freq 160→40 Hz (high to low), opposite of up-chirp. Analog signal, DSP never sees it.
⑤ Conjugate down-chirp c*(t): Baseband I/Q ★
Locally stored complex sequence c*[n]. I is same as up-chirp, Q is negated = mathematical conjugate (phase inversion). c* bandwidth is also ±BW/2; when aliasing: dashed=exact c*(t), solid=sinc reconstruction (undersampled aliased version), which is what the multiplier actually operates on.
I = cos(φup) Q = −sin(φup) I[n] Q[n]
⑥ Product r(t)·c*(t): Passband Reference Waveform
Continuous-domain product, frequency no longer varies → passband pure cosine. Continuous reference, DSP never sees it.
⑦ Product y[n] = r·c*: Baseband I/Q ★ (FFT input)
When s≠0 there are actually two different frequencies (s·Δf and (s−N)·Δf) — that's why FFT must use N_SYM=24 chip samples for DFT: the two segments resonate on the 24-pt grid = single peak. When aliasing: dashed=true continuous signal, colored solid=clean sinc reconstruction. SNR<∞: gray solid=noisy reconstruction, shows how noise distorts the FFT input.
I = Re{y[n]} Q = Im{y[n]} I[n] Q[n]

📐 Discrete-Domain Step-by-Step (what DSP actually does)
Step 0 ▸ Hardware IQ Downconversion + ADC (not done in software) Antenna s_RF(t) = Re{r_BB(t)·e^{j2πF_c·t}} Mixer 1: s_RF × cos(2πF_c·t) → LPF → I_analog(t) Mixer 2: s_RF × (−sin(2πF_c·t)) → LPF → Q_analog(t) Dual-channel ADC → r[n] = I[n] + j·Q[n] n = 0,1,…,N_samp−1 Step 1 ▸ Pre-generate conjugate down-chirp (lookup table, not real-time) φ_up[n] = 2π·(−BW/2·n/Fs + BW/(2T)·(n/Fs)²) c*[n] = cos(φ_up[n]) − j·sin(φ_up[n]) ← Q negated = mathematical conjugate Step 2 ▸ Point-wise complex multiply (de-chirp core, only N_samp complex multiplications) r[n] = a + j·b c*[n] = c − j·d (d = sin(φ_up[n])) y[n] = r[n]·c*[n] = (ac+bd) + j·(bc−ad) ← phase subtraction Step 3 ▸ DFT (N = BW·T points, one per symbol) Y[k] = Σ y[n]·e^{−j2πkn/N} k = 0,…,N−1 Peak bin k* → decoded symbol ŝ = k* (N_samp may ≠ N; align via zero-pad/resample) Why can't we drop the Q channel? (Negative frequencies & analytic signal) cos(φ) is a real even function: cos(+φ) = cos(−φ), spectrum is two-sided symmetric → Cannot distinguish e^{+jφ} (up-chirp) from e^{−jφ} (down-chirp) I+jQ = e^{jφ} is an analytic signal, single-sided spectrum (no negative-freq aliasing) → After DFT: k=5 (up-chirp) and k=N-5 (down-chirp) are separate, no collision About IQ Hardware (real LoRa chip) RF = 868 MHz, BW = 125 kHz → baseband ±62.5 kHz Inside chip: analog IQ mixing (868MHz × cos/sin) → analog LPF → ADC ADC rate = ~1 Msps (far above Nyquist 125kHz), then digital decimation down to BW Final baseband complex sample rate = BW = 125 kHz, DSP processes N = 2^SF points/symbol
