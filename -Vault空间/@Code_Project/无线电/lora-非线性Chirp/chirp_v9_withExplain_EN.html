📡 Channel Immunity (Multipath & Doppler)
Multipath: 100m reflection true delay = 0.33μs, at 500kHz sample rate = only 0.17 samples → rounds to 0. Reflection overlays directly into main signal; matched filter processes it together, no ISI. Freq-selective fading requires delay > 1/BW = 2μs (>600m path diff), typically only in urban high-rise environments. CSS long symbols are naturally immune to multipath.
Doppler: v=100m/s @2.4GHz gives fd=800Hz. After downconversion to 500kHz baseband, 800Hz offset = 0.16%. Scaled to 500Hz simulation BW = 0.8Hz shift = 0.1 FFT bins, completely buried within the chirp correlation main lobe (≈2 bins wide). CSS is inherently robust to Doppler, no compensation needed.
Phase Offset: In traditional linear chirp "conjugate multiply + FFT", phase offset has zero impact — it synchronously affects the product phase only.
Frequency Offset: In traditional linear chirp, it synchronously shifts the product frequency. In nonlinear chirp — not yet calculated. Corrected via preamble AFC.
The Real Threat: 2.4GHz band WiFi/Bluetooth narrowband interference (continuous CW) and harmonic EMI impulse noise — these are the primary adversaries for drone SDR links. See jamming analysis below.

📊 Nonlinear Chirp CSS: LPI Value & Engineering Tradeoffs
Why does nonlinear Chirp have LPI advantage?
Standard LoRa (linear Chirp) has an analytic frequency trajectory f(t) = FL + (FH-FL)·t/M — any SDR can identify it with one dechirp+FFT.
Perlin/nonlinear Chirp trajectory is determined by a seed. Without the seed, no analytic dechirp can be constructed, achieving LPI (Low Probability of Intercept). An adversary only sees a diffuse blob on the wideband spectrum, indistinguishable from noise — this is true LPD (Low Probability of Detection).
Cost: The receiver must store a full set of templates (NA×M samples); cannot use the `conjugate-multiply then FFT` shortcut. Acceptable when NA is small (≤16).

Increasing M → Higher Sensitivity (Processing Gain)
Processing gain = 10·log₁₀(M), independent of NA. There are two paths to larger M:
Path How M Increases Symbol Duration Cost
A. Faster ADC FS×2, T unchanged → M×2 Unchanged ADC power/cost ×2
B. Longer Symbol FS unchanged, T×2 → M×2 ×2 (longer) Data rate ÷2, latency ×2
Both paths yield +3dB. The choice: spend ADC money or spend time.
M Processing Gain Duration (BW=500kHz real) Notes
64 (SF6) +18.1 dB 128 μs
128 (SF7) +21.1 dB 256 μs ← Current
256 (SF8) +24.1 dB 512 μs M×2 = +3dB
512 +27.1 dB 1.024 ms
1024 +30.1 dB 2.048 ms
Doubling both M & NA vs. doubling M alone:
Action Sensitivity bits/sym Compute Reason
M×2 (NA unchanged) +3dB No change ×2 Integration time ×2, noise ± cancellation more effective
NA×2 (M unchanged) 0 (no impact) +1 bit ×2 NA is completely unrelated to processing gain; it only expands the symbol alphabet
M×2 and NA×2 +3dB +1 bit ×4 Gain comes from M only; NA contributes just 1 bit/sym, at 4× compute cost
Conclusion: Sensitivity is determined solely by M (integration sample count); NA (symbol template count) contributes nothing to sensitivity. Want better sensitivity → increase M. Want higher-order modulation → increase NA (but NA>16 begins to defeat the purpose of nonlinear chirp).
⚠️ Counterintuitive: Increasing BW (M fixed) → Sensitivity DECREASES! SX1276 confirms it
SX1276 datasheet measurements: SF7/BW125 → -124dBm, SF7/BW500 → -118dBm (4×BW = -6dB sensitivity, exactly 10log₁₀(4)=6dB).
Source: Semtech SX1276/77/78/79 Datasheet Rev.7, Table 16 Receiver Characteristics (product page for latest version)

Root cause (RF systems): Total noise power N = N₀ × BW. BW×2: ADC sample rate ×2, symbol duration T = M/BW halves, integration time shortens, SNR drops 3dB; plus wider ADC bandwidth admits more noise. Net effect: BW×k → sensitivity -10log₁₀(k) dB.

BW increase's real value (besides sensitivity):
✅ Higher data rate! CSS orthogonality requires Tₛ = M/BW. BW×2 → Tₛ halves → more symbols per second → data rate doubles. ADC sample count M stays the same, but clock frequency scales with BW; symbols complete in less time.
BW Change M Change Sensitivity Data Rate LPI Level CW Resilience ADC Req.
BW×2 M unchanged -3dB (worse) ×2 (better) ↑↑ (wider spread) ↑↑ (more freq diversity) 2× faster
BW×2 M×2 Unchanged (proportional) ×2 (better) ↑↑ ↑↑ 2× faster, dur.×1
BW unchanged M×2 +3dB (better) ÷2 (worse) Unchanged Unchanged Unchanged, dur.×2
Conclusion: Widening BW's real value is data rate↑, LPI (lower PSD), and frequency diversity (NBI resilience) — but sensitivity drops. To improve sensitivity you must increase M (faster ADC or longer symbol).

Scenario Signal S Noise N Source SNR Limiting Factor Strategy

Root cause: N = k·T·B·F. Near-field: S is strong, external interference far exceeds thermal noise (F is irrelevant). Far-field: interference vanishes, N reverts to thermal noise, every 1dB of NF costs 1dB of sensitivity.
For our drone tactical link: the primary BLOS scenario is far-field low-interference — LNA quality and M size are top priority. Urban near-field backup links rely more on interference resilience (processing gain + spreading bandwidth).
📶 IQ Downconversion & Template Matching
Standard LoRa's conjugate multiply performs IQ demodulation and dechirp simultaneously — two steps in one. Our template matching separates them:
Stage Standard LoRa Our Nonlinear Chirp
Downconversion IQ dual-channel (required) IQ dual-channel (still required — image rejection)
Demod Core r(t)·s*(t) (conjugate multiply, uses Q path) → |FFT| |Σ(I_r+jQ_r)·(I_t−jQ_t)| (complex dot product, take magnitude)
Effect of phase offset φ Eliminated by magnitude, no impact Eliminated by complex magnitude, no impact (real dot product would multiply by cosφ!)
Primary source of freq offset Crystal accuracy (±20ppm → ±18kHz @915MHz) far exceeds Doppler (drone 20m/s → 61Hz). Requires preamble AFC compensation.
Note: The animations on this page demonstrate in real baseband (implying “IQ downconversion done, phase aligned”). In hardware, store complex templates I_t+jQ_t, perform complex dot product and take magnitude — phase offset is automatically eliminated.

📝 System Design Memo · Perlin-CSS Tactical Link Engineering Decisions
① Naming & Positioning
Chirp ⊃ CSS (Chirp Spread Spectrum) ⊃ LoRa (Semtech linear CSS patent)
Recommended name for this scheme: Perlin-CSS or Nonlinear CSS
Do NOT call it "LoRa" (cannot dechirp+FFT demodulate; incompatible with Semtech patent)

Positioned closest to APART (Adaptive Phonon-Activated Radio Transceivers) philosophy: high LPI priority; seed = key; without seed, waveform cannot be replicated. Standard LoRa linear chirp is identifiable by any SDR; this scheme is not.
② No Pilots Needed — Preamble Handles All Sync
WiFi (OFDM) needs pilot subcarriers — each subcarrier in a multi-carrier system requires independent channel estimation.
CSS is single-carrier spread spectrum; the preamble (a few repeated chirps) handles all synchronization:
  • AGC settling (receive gain lock)
  • Timing sync (preamble autocorrelation peak → symbol boundary)
  • AFC frequency correction (estimate crystal offset and compensate)
Data symbols need no inserted pilots.

Perlin preamble scheme (recommended): Both sides agree on preamble_seed=0; send N repeated Perlin chirps. AFC scans candidate Δf offsets for max correlation peak (with TCXO, range is only ±500Hz — negligible cost). Advantage: no linear chirp signature anywhere, preamble is also unidentifiable — complete LPI.
Primary freq offset source: crystal accuracy (±20ppm → ±18kHz @915MHz) far exceeds Doppler (drone 20m/s → 61Hz, negligible).
③ Duplex Design vs. Random Transmit Timing
Random transmit timing has a duplex problem: if both ends are random, bidirectional comms can't be guaranteed. Options:
Scheme Scenario LPI Level Complexity
2× Simplex Units (separate uplink + downlink) Abundant hardware High Low
Call-Response Trigger (drone sends beacon, controller replies, enters RX) Asymmetric data (more control, less telemetry) Medium (beacon is periodic) Medium
Pseudorandom Frame-Interval TDD (negotiated PR sequence for tx/rx alternation) Symmetric bidirectional, high LPI High (irregular intervals) Medium
GPS PPS Hard-Sync TDD (integer-second alignment, agreed frame structure) Drone (with GPS) primary scenario Highest (preamble can be omitted) Low (GPS already present)
GPS PPS scheme details: Both sides agree: "at millisecond N of each integer second, start frame K". Sync without any preamble. The adversary sees no preamble energy signature at all — the most thorough LPI.
No-GPS fallback: TCXO + initial handshake (one-time sync at power-on) + crystal drift compensation (1ppm TCXO → 3.6ms drift/hour, needs periodic re-sync).
Frame length/interval/sequence should be designed per actual traffic (drone telemetry vs. robot dog bidirectional vs. other); no universal optimum exists.
④ Is Perlin Chirp Still Needed with DSSS?
With DSSS, the PN code already provides:
  • Spectrum spreading (noise-like PSD)
  • Multipath resilience (high chip time resolution)
  • Multiple access (different PN codes are orthogonal)
Perlin chirp's additional value: The frequency trajectory cannot be analytically replicated (seed = key). Even knowing it's CSS, an adversary cannot dechirp. If the DSSS PN code structure is cracked, it can still be despread.

Conclusion: Pure DSSS implementation is hard (chip-level sync). If resource-constrained, Perlin-CSS alone is sufficient. DSSS+Perlin stacked is the strongest LPI, but doubles hardware cost — classify as Phase-2 enhancement option.
DSSS HW challenges: chip-level clock sync (an order of magnitude harder than symbol-level), chip rate must be integer ratio with ADC clock, dual-layer processing chain adds latency.
⑤ FEC or Longer M?
CSS already has built-in time-diversity integration; M already acts as soft-decision error correction. FEC in chirp has poor cost/benefit:
Method Gain Cost
M doubled (256→512) +3dB Data rate ÷2; faster ADC or 2× duration
FEC rate 1/2 ≈3–6dB (coding gain) Data rate ÷2; encode/decode compute ↑↑
Verdict: FEC is viable. Consider when link margin is insufficient and M cannot be increased.
If using FEC, recommend lightweight Hamming(7,4) or Reed-Solomon. Avoid Turbo/LDPC (embedded compute overhead not worth it).
⑥ Inter-Frame FHSS Scheme
No large frequency jumps within a symbol (chirp continuity required); only inter-frame hopping. After IQ downconversion, the frequency center is normalized — this can be omitted in simulation but must be implemented in hardware.

Most random ≠ Best LPI: Pure random hopping still statistically exposes the operating band (wideband scan finds energy at specific frequencies).
LPI-optimal hopping should:
  • Cover all available frequencies uniformly (no preference exposure)
  • Use a pseudorandom sequence (seed secret = hop key)
  • Keep dwell time short (per-frame energy extremely low, single frame hard to detect)
  • Hop rate > adversary scan rate
Costas array hopping (military theoretical basis): Mathematically guarantees any two time slots and any two frequencies meet at most once — both uniform and orthogonal.
Multi-device coexistence: Different devices use different FHSS seed sequences; occasional collisions handled by CRC frame drop — simpler than coordination overhead.
Recommended band: 902–928MHz ISM (26MHz hopping range), BW=500kHz, ≈52 independent channels.
⑦ OCDM (Orthogonal Chirp Division Multiplexing)
Analogous to OFDM but with all subcarriers replaced by chirps — chirps with different starting offsets are mutually orthogonal; multiple data streams transmit simultaneously in the same band.

Theoretical advantage: Chirp's time-frequency duality provides inherent robustness to multipath and time-frequency offset, outperforming OFDM against frequency-selective fading. ETH Zurich and others have academic research on this.
Practical cost: Receiver must perform independent matched filtering per subcarrier; complexity = NA × number of subcarriers. Difficult to implement in embedded systems.
Verdict: Little value for drone tactical links (too complex). Archive as the ultimate form of the chirp family. If FPGA acceleration matures, could be explored for high-throughput scenarios.
