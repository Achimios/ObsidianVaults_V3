📊 非线性Chirp CSS 原理 · LPI价值与工程权衡
为什么非线性Chirp有LPI优势？
标准LoRa（线性Chirp）的频率轨迹是解析的 f(t) = FL + (FH-FL)·t/M，任何SDR只需做一次dechirp+FFT即可识别。
Perlin/非线性Chirp的轨迹由seed决定，不知道seed就无法构造解析dechirp，达到LPI (Low Probability of Intercept)。敌方只能看到宽带频谱上的一团扩散信号，无法区分信号与噪声——这才是真正的LPD (Low Probability of Detection)。
代价：接收端必须存全套模板（NA×M个样本），无法用`共轭相乘再FFT`加速。当NA较小时（≤16）代价可接受。

处理增益 = 10·log₁₀(M)，与NA无关。M的来源有两种：

两种路径都给+3dB。区别是花ADC钱还是花时间。

⚠️ 反直觉：BW增大（M不变）→ 灵敏度反而下降！SX1276 实测印证
SX1276 数据手册实测：SF7/BW125→-124dBm，SF7/BW500→-118dBm（4×BW=-6dB灵敏度，恰好 10log₁₀(4)=6dB）。
数据来源：Semtech SX1276/77/78/79 Datasheet Rev.7，Table 16 Receiver Characteristics（产品页面可下载最新版）

根本原因（RF系统）：总噪声功率 N = N₀ × BW。BW×2，ADC采样率×2，符号时长 T = M/BW 减半，积分时间变短，SNR下降3dB；同时更宽ADC带宽混入更多噪声。净效果：BW×k → 灵敏度 -10log₁₀(k) dB。

BW增大的真正价值（非灵敏度）：
✅ 数据率提升！ CSS正交性条件要求 Tₛ = M/BW。BW×2 → Tₛ减半 → 每秒发更多符号 → 数据率翻倍。 ADC采样点数M不变，但时钟频率跟着BW等比加快，符号在更短时间内完成。

BW×2 M不变 -3dB（变差） ×2（提升） ↑↑（更宽扩散） ↑↑（更多频率分集） 快×2
BW×2 M×2 不变（等比） ×2（提升） ↑↑ ↑↑ 快×2，时长×1
BW不变 M×2 +3dB（提升） ÷2（降低） 不变 不变 不变，时长×2
结论：扩宽BW的真正价值是数据率↑、LPI（更低PSD）和频率分集（抗窄带干扰），但灵敏度下降。要提灵敏度必须增大M（更快ADC或更长symbol）。

根本因：N = k·T·B·F。近距时S强，外部干扰远超热噪声（F无意义）；远距时干扰消失，N回归热噪声，F每差1dB灵敏度就差1dB。
对我们无人机战术链路：BLOS主场景属于远距低干扰，LNA质量和M的大小是第一优先级。城区近距备用链路则更依赖抗干扰能力（即处理增益+展频宽度）。

📝 系统设计讨论备忘录 · Perlin-CSS 战术链路工程决策全记录
① 命名层次与方案定位
Chirp ⊃ CSS（Chirp Spread Spectrum）⊃ LoRa（Semtech 线性CSS专利）
本方案推荐名称：Perlin-CSS 或 非线性CSS
不可说"LoRa"（无法 dechirp+FFT 解调，与 Semtech 专利不兼容）

定位最接近 APART (Adaptive Phonon-Activated Radio Transceivers) 风格：高LPI优先，种子即密钥，无种子无法复现波形。 标准LoRa线性chirp任何SDR均可识别，本方案不行。
② 不需要导频，前导码包办同步
WiFi (OFDM) 需要导频子载波——多子载波系统每个子载波需独立信道估计。
CSS 单载波扩频，前导码（几个重复chirp）承担全部同步功能：
  • AGC 稳定（接收增益锁定）
  • 定时同步（前导码自相关峰 → symbol 边界）
  • AFC 频率校正（估计晶振偏差并补偿）
数据符号里不需要插入额外导频。

Perlin 前导码方案（推荐）：双方约定 preamble_seed=0，发 N 个重复 Perlin chirp。 AFC 需对候选 Δf 扫描找最大相关峰（有 TCXO 则范围仅 ±500Hz，代价很小）。 优势：全程无线性 chirp 特征，前导码也不可识别——彻底 LPI。
频率偏移主因：晶振精度（普通 ±20ppm → ±18kHz @ 915MHz）远大于多普勒（无人机 20m/s → 61Hz，可忽略）。
③ 双工设计 vs 随机发射时机
随机发射时机有双工问题：若两端都随机，无法保证双向通讯。可选方案：
方案 适用场景 LPI程度 复杂度
2对单工设备（上行+下行各一套） 硬件资源宽裕 高 低
呼叫-应答触发（无人机发"yy"信标，遥控器回"听到"进入接收） 非对称数据量（遥控多，回传少） 中（信标有规律） 中

GPS PPS 硬同步 TDD（整秒对齐，约定帧结构） 无人机（有GPS）主战场景 最高（可省前导码） 低（GPS已有）
GPS PPS 方案详解：双方约定"每整秒第N毫秒开始第K帧"，无需发前导码即可同步。 敌方连"有前导码"的能量特征都看不到——最彻底的 LPI。
无GPS备用：TCXO + 初始握手（上电一次对时）+ 晶振漂移补偿（1ppm TCXO，1小时漂移 3.6ms，需定期重同步）。
帧长/间隔/顺序应根据实际业务设计（无人机回传 vs 机械狗双向 vs 其他）；不存在普适最优方案。
④ DSSS 存在时是否还需要 Perlin chirp？
有 DSSS 时，PN 码已提供：
  • 频谱扩散（类噪声 PSD）
  • 抗多径（chip 时间分辨率高）
  • 多址复用（不同 PN 码正交）
Perlin chirp 的额外价值：频率轨迹不可解析复现（seed 即密钥），即使知道是 CSS 也无法 dechirp。 DSSS 的 PN 码结构若被破解仍可解扩。

结论：纯 DSSS 实现难度高（chip 级同步），若资源受限，Perlin-CSS 单独已够用； DSSS+Perlin 叠加是最强 LPI，但硬件成本翻倍，标注为 Phase-2 增强选项。
DSSS 硬件难点：chip 级时钟同步（比 symbol 级难一个数量级）、chip rate 与 ADC 时钟需整数比、双层处理链延迟增加。
⑤ FEC 还是更长的 M？
CSS 已自带时间分集积分，M 已等效软判决纠错。FEC 在 chirp 里性价比差：
方法 增益 代价
M 翻倍（256→512） +3dB 数据率÷2，ADC更快或时长×2
FEC 码率 1/2 ≈3-6dB（编码增益） 数据率÷2，编解码计算量↑↑？
结论：FEC好像也行，在链路余量不足且无法增大M时考虑。
若要 FEC，推荐轻量的 Hamming(7,4) 或 Reed-Solomon， 不推荐 Turbo/LDPC（嵌入式算力开销不值得）。
⑥ 帧间 FHSS 方案
符号内不可大幅跳变（chirp 连续性），只有帧间跳频。IQ下变频后频率中心已归一化，仿真里可省略，但硬件必须实现。

最随机 ≠ LPI 最优：纯随机跳频仍会在统计上暴露工作频段（宽带扫描找有能量的频点）。
LPI 最优跳频应：
  • 均匀覆盖全部可用频点（不暴露偏好频率）
  • 伪随机序列（seed 保密 = 跳频密钥）
  • 驻留时间短（每帧能量极低，单帧难被检测）
  • 跳频速率 > 敌方扫频速率
Costas 阵列跳频（军用理论基础）：数学上保证任意两时隙任意两频点最多相遇1次，既均匀又正交。
多设备共存：不同设备用不同 seed 的 FHSS 序列，少量碰撞靠 CRC 丢帧处理，比协调开销更简单。
推荐频段：902-928MHz ISM（26MHz 内跳），BW=500kHz，约 52 个独立频道。
⑦ OCDM（Orthogonal Chirp Division Multiplexing）
类比 OFDM 但子载波全换成 chirp——不同起始偏移的 chirp 互相正交，多个数据流同时在同一频带传输。

理论优势：chirp 的时频对偶性使其对多径和时频偏移天然鲁棒， 比 OFDM 更抗频率选择性衰落。苏黎世联邦理工（ETH Zurich）等有学术研究。
实际代价：接收端需对每个子载波独立匹配滤波，复杂度 = NA × 子载波数， 嵌入式难以实现。
结论：对无人机战术链路意义不大（过于复杂），作为 chirp 系族终极形态存档。 若未来 FPGA 加速成熟，可作为高吞吐量方向研究。

💡 发散性思考与可操作后续步骤
① 仿真与现实之间的差距
本仿真使用实数基带点积，隐含假设：IQ下变频已完成、相位已对齐、AFC已校正。真实硬件会增加：(a) IQ不平衡（I/Q通道的增益/相位失配 → 镜像抑制不完美），(b) ADC量化噪声（12-bit ENOB → ~11有效位），(c) 功放非线性（为效率而推到近饱和时产生AM-AM/AM-PM失真），(d) LNA噪声系数（典型的2-4dB直接从链路预算中扣减）。
行动：首次HackRF测试应测量带真实IQ不平衡的EVM，对照仿真基线校准这些差距。
② Perlin噪声参数调优
目前M=128使用10个octave的Perlin噪声。更少octave → 更平滑的chirp → 正交性降低但同步更容易。更多octave → 更锐利的轨迹 → 正交性更好但对定时抖动更敏感。
行动：在仿真器中添加"octaves"滑块（目前硬编码为10）。扫描2-20个octave，测量互相关矩阵非对角泄漏 vs. 定时误差容忍度。
③ 模板预计算与存储策略
NA=8 M=128时模板需要8×128=1024个浮点数(4KB)。但如果每帧轮换seed，模板必须实时重新生成。方案A：双缓冲——在当前帧期间为下一个seed预计算所有模板。方案B：在MCU上用定点数实现快速Perlin生成器（STM32H7 DSP约2个周期/lerp）。
行动：在目标MCU上对Perlin生成速度 vs. 内存读取速度进行基准测试。
④ Perlin之外的替代非线性函数
Perlin噪声不是唯一选择。Logistic映射：x[n+1]=r·x[n]·(1-x[n]), r≈3.99达到混沌——计算更简单。Lorenz吸引子：3个耦合ODE，动力学更丰富。稀疏跳频chirp：将M分段，段内跳频。行动：添加"chirp函数"下拉菜单，将logistic映射作为快速测试案例。比较LPI指标（PSD平坦度、循环平稳特征）。
⑤ 敌方实际会怎么做？
拥有SDR的对手会尝试：(1) 循环平稳分析：CSS在符号速率=BW/M处存在固有的循环平稳性——即使非线性chirp也能检测到。(2) 长积分能量检测：如果发射功率超过噪底+处理增益，长时间平均会暴露存在（但不暴露内容）。(3) 基于ML的chirp分类：在已知chirp类型的频谱图上训练CNN → 即使没有seed也可能指纹识别Perlin-CSS。缓解措施：在不同seed家族间跳变、发送假传输、使用最小功率控制。
⑥ 硬件测试路线图（按顺序）
阶段1（桌面）：STM32H7 + AT86RF215评估板。测试：基带IQ环回，测量EVM vs. SNR。确认实时模板匹配时序。阶段2（有线）：两块板，中间接衰减器（无天线）。扫描SNR -20到+20dB，测量BER vs. 理论值。阶段3（OTA，开阔地）：加天线，100m-1km距离。测量多径免疫性、距离 vs. BER。阶段4（干扰）：用HackRF作为干扰源引入CW/脉冲干扰，测试韧性。
⑦ 与现有SDR框架的集成
GNURadio：编写Python "Perlin CSS调制器"模块（输出IQ流）+ "Perlin CSS解调器"模块（模板匹配）。可直接复用HackRF osmocom source/sink。行动：先原型化调制器模块——只需调用本仿真器中的ifPerlin(seed)和ifToWave()，封装为GNURadio同步模块。
⑧ 品质因数：LPI评分
需要一个量化比较LPI的方法。建议：LPI评分 = PSD平坦度 × seed空间大小 × 循环平稳峰值抑制。PSD平坦度 = std(PSD)/mean(PSD)（越低越好）。Seed空间大小 = 2^(seed比特数)（32-bit seed = 40亿种可能）。循环平稳抑制 = 符号速率处相关峰 / 噪底。行动：在仿真中添加自动计算的LPI评分指示器。
⑨ 基于Seed正交性的多用户接入
Perlin-CSS可以像CDMA一样工作：为每架无人机分配不同的seed族。不同seed的chirp之间互相关保持较低（但不保证为零——不像Walsh码那样严格正交）。优势：没有固定码本可被破解。核心问题：多少个用户可以同时共享频段？快速估算：对M=128的随机信号，互相关≈1/√M≈0.088=−21dB（低于自相关峰值）。支持约4-8个同时用户，超出后性能下降。行动：添加多用户仿真模式——运行4个并行相关器，绘制BER vs. 用户数。
⑩ 前导码是LPI最薄弱的环节
前导码（同步序列）在每帧数据前明文发送——是最容易被利用的模式。标准LoRa使用重复上啁啾，极易检测。即使使用Perlin啁啾，如果前导码结构固定，对手依然可以指纹识别。方案A：用seed派生的伪随机相位偏移加密前导码——接收方知道seed可以反向解密。方案B：无专用前导码；用已知明文的载荷头部作为隐式同步。方案C：噪声骑行——仅在信道存在环境噪声突发时发送。行动：先实现方案A——几乎零额外复杂度，却完全打破前导码可检测性。
