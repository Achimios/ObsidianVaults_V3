## 一、为什么要做这个实验

上一轮对话里，豆包在解释 LoRa 解调时说了一句话：

> "传统方案是将接收信号与本地基准 Chirp 做**滑动内积**"

这句话是**错的**。正确操作叫 **Dechirping + FFT**。

两者表面相似，但搞混了就会在实现和理解上走弯路。这个实验用代码直接证伪，顺便生成可视化让眼睛也信服。

---

## 二、LoRa 到底在干什么？（从零说起）

### 什么是 Chirp

Chirp 是"频率随时间变化的正弦波"，像救护车经过时声音的音调变化。
LoRa 用的是**线性 up-chirp**：频率从低往高**线性**扫，扫完一个带宽（比如 125 kHz）。

### 怎么编码符号？

LoRa 里，`SF=7` 代表 $2^7 = 128$ 个不同符号（0 ~ 127）。
每个符号 = 一个 up-chirp，区别在于**从哪个频率开始扫**。

符号 0：从最低频开始扫
符号 42：从中间某个频率开始扫（扫到头后折回）
符号 127：从快到最高频的位置开始扫

数学上，符号 $s$ 的 chirp 相位是：
$$\phi(t) = \frac{2\pi}{N} \left( s \cdot t + \frac{t^2}{2} \right), \quad t = 0, 1, \ldots, N-1$$

其中 $N = 2^{SF}$ 是每个符号的采样点数。

## 三、正确的解调流程（三步）

### 步骤 ① 接收信号

接收到的就是发射端发的 chirp（加上噪声）。  
时域看上去是"频率越来越高的正弦波"，什么信息都看不出来。

### 步骤 ② Dechirping（去线性调频）

把接收信号乘以**基准 chirp 的共轭**（即符号 0 的 chirp 取复共轭）：

$$y(t) = r(t) \cdot \overline{c_0(t)}$$

数学上发生了什么？两个 chirp 相乘，二次项（$t^2/2$）消掉了，剩下：

$$y(t) = e^{j \frac{2\pi}{N} s \cdot t}$$

这是一个**纯单频复正弦波**，频率正好是 $s/N$（即符号值除以符号数）。  
时域从"频率在变"变成了"频率固定"——这就是"去线性调频"的意义。

> **这不是"滑动内积"**：没有窗口在时间轴上滑动，而是整个信号一次性做共轭相乘。

### 步骤 ③ FFT（找峰值）

对 $y(t)$ 做快速傅里叶变换，在频域找幅度最大的 bin（频率格子）。  
由于 $y(t)$ 是频率为 $s/N$ 的单频信号，**峰值所在的 bin 编号就等于符号值 $s$**。

$$\hat{s} = \arg\max_k \left| \text{FFT}(y)[k] \right|$$

搞定。

## 四、实验结果

### 无噪声验证（100% 通过）

| 发送符号 | 检测结果 | 正确？ |
|---------|---------|--------|
| 0 | 0 | ✓ |
| 1 | 1 | ✓ |
| 32 | 32 | ✓ |
| 64 | 64 | ✓ |
| 126 | 126 | ✓ |
| 127 | 127 | ✓ |

### SNR 扫描结果（100 符号/点）

| SNR | 正确率 |
|-----|--------|
| -20 dB | 10% |
| -14 dB | 56% |
| -12 dB | 77% |
| **-10 dB** | **100%** |
| -8 dB 及以上 | 100% |

**结论：99% 门限 SNR ≈ -10 dB**

这个值与 LoRa 技术规格书中的理论灵敏度（SF7 约 -123 dBm，换算成信噪比约 -10 dB 量级）高度一致，说明仿真是可信的。

```
lora_dechirp.py
│
├── gen_upchirp(symbol, N)   ← 生成 chirp 信号
│     └── 用相位公式生成复数时域序列
│
├── demodulate(rx, N)        ← 核心解调（3 行代码）

      └── 保存 lora_snr_curve.png
```

## 六、与豆包描述的区别，用一句话

| | 豆包说的 | 实际 |
|---|---|---|
| **操作名称** | 滑动内积 | 共轭相乘（Dechirping） |
| **滑动窗口？** | 有（窗口在时间轴滑动） | 无（整个符号一次性计算） |
| **本质** | 时域相关（卷积型） | 混频器（前端解调） + 频域峰值检测 |
| **数学等价性** | 有（在某些条件下等价于匹配滤波） | 匹配滤波器的高效实现 |

> 注：数学上 dechirping+FFT 是 chirp 匹配滤波器的高效形式，和互相关有深层联系，但"滑动内积"这个说法会误导实现。
