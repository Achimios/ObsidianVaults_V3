# TIME WANDERER — 机制技术规格

## 1. 物理世界观框架（Gemini 方案落地版）

### 1.1 为什么人类被"锁"在 3D？

宇宙的时空结构是一个**各向异性晶体**（anisotropic crystal）。在 $w$ 轴（第四维）方向，时空具有极高的"虚复阻抗"。

$$Z_w \gg Z_{xyz}$$

**坡印廷向量**（能流密度）在 $w$ 轴上受到极强阻尼：能量倾向于在 $x$-$y$-$z$ 方向流动，几乎无法"漏"进第四维。

- **结果**：宏观物体被粘在特定的 $w$ 切片上，跨越 $w$ 需要克服指数级能量壁垒。
- **日常经验**：人类永远感知不到 $w$ 轴存在，就像你不会感受到自己在 $w$ 方向的"厚度"——它的导电率太低了。

**游戏设计含义**：  
大多数 NPC 无法自主改变 $w$ 值。地图、建筑、物品在 $w$ 轴上"固定"，从玩家视角看是静止的背景层。

---

### 1.2 超维度长丝（Hyper-filament）：质量的真正定义

所有有质量的物体，在 4D 空间中并非"点"，而是贯穿 $w$ 轴的**长丝**。

$$\text{可见质量} = \text{长丝在 } w_0 \text{ 切面的截面能级}$$

**长丝模型的直接推论**：

| 性质 | 解释 |
|------|------|
| $m_w$（w 惯性质量）| 长丝的**刚度**——刚度越高，改变 $w$ 越难 |
| 普通人 $m_w$ 大 | 长丝刚，像铁棍，不能弯曲 → 不能移动 $w$ |
| Wanderer $m_w \approx 0$ | 长丝极细极柔，像一根线 → 几乎无代价移动 |
| 同伴跟随延迟 | 长丝耦合：被 Wanderer 拖拽时，自身刚度决定响应速度 |

**拖拽效应**（同伴跟随的物理解释）：  
当 Wanderer 移动时，他的长丝拖动周围的长丝——就像把一根针从书里拔出时，所有书页上的针孔会一起位移。刚度高（$m_w$ 大）的物体延迟大，刚度极高的物体（建筑、岩石）几乎不跟随。

---

### 1.3 遗迹维度：高导通区

$$w \in [0.5, \infty)$$

"遗迹区"是 $w$ 轴上**阻抗异常低**的区域。坡印廷向量不再被锁在 3D，能量向四周发散——物质的分子键无法维持，导致：

- 物体逐渐"流向"相邻 $w$ 层（视觉：材料有半透明质感，边缘模糊）
- 生命体在这里存活需要稳定的"锚定源"（节点）
- 遗迹层本质是**三百年前时空晶体的一次局部软化**，由 Princess 触发，被她的意识维持

## 2. Wanderer 的感知系统

### 2.1 感知半径 $\Delta w$

Wanderer 能感知以自身 $w$ 值为中心、半径 $\Delta w$ 内的信号。

$$\text{感知范围} = [w_0 - \Delta w, w_0 + \Delta w]$$

**初始值**：$\Delta w = 0.3$  
**最终值**（满级）：$\Delta w = 0.8$  
**提升方式**：积累"先知"称号（感知类操作经验）

---

### 2.2 感知扫描（主动扫描）

**触发**：按 R 键  
**效果**：向外发出一个球形感知脉冲，探测所有感知范围内的信号源  
**代价**：扫描会被**晶格牧师的被动探测网络记录**

信号在感知场中的表现：

$$\text{信号强度} \propto \frac{1}{|\Delta w|^2 + |\Delta r|^2}$$

其中 $\Delta w$ 是 w 轴距离，$\Delta r$ 是 xyz 平面距离。w 轴距离比平面距离的衰减更快（因为晶格各向异性），所以在同等物理距离下，$w$ 方向的信号更弱。

## 3. 指示器系统（UI 设计规格）

### 3.1 指示器物理原型

指示器是**能流场偏转探测仪**，非传统罗盘。

它探测的是 Princess 意识节点释放的"能流扰动"——那是时空晶体在节点附近发生微小软化时，坡印廷向量局部偏转产生的可探测信号。

### 3.2 指示器视觉设计（Godot UI 实装目标）

```
┌─────────────────────────────────┐
│                                 │
│    ·  ·  ·  ◉ ←  ·  ·  ·        │  ← 3D 方向指示（罗盘圆环）
│                                 │
│   ▼  w轴方向：更深               │  ← w 轴偏移指示（竖条）
│                                 │
│   ≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋      │  ← 噪声可视化（波纹）
│                                 │
└─────────────────────────────────┘
```

**三个独立子组件**：

| 组件 | 功能 | 视觉 |
|------|------|------|
| 方向环 | xyz 平面方向 | 圆环，信号点在环上旋转，近则亮 |
| 深度条 | w 轴上下偏移 | 竖条，点在条上滑动，亮度 = 信号强度 |
| 噪声层 | 干扰可视化 | 波纹强度随反射器密度变化 |

**噪声机制**：
- 纯信号：波纹微弱，方向稳定
- 反射器干扰：波纹强烈，方向随机震荡
- 萨奇加入后：噪声层出现一个"校正叠加层"，反射器干扰被部分消去

---

### 3.3 极光视觉效果（靠近节点时）

灵感来自 Gemini 的建议："能流在各向异性介质中偏转产生的能流溢出"。

**触发条件**：与节点 xyz 距离 < 50m 且 $|\Delta w| < 0.3$  
**效果**：屏幕边缘出现蓝白色极光状光晕，强度随距离增加  
**声效配合**：低频共鸣，非线性 Chirp 信号（参考 $z_非线性Chirp解调/$ 研究）

Godot 实装：`CanvasLayer` + `ShaderMaterial`，极光着色器参考：


## 4. 跨维度攻击（超空间侧击）

### 4.1 物理原理

从 $w$ 轴方向发动的攻击，绕过目标在 $x$-$y$-$z$ 方向部署的所有防御。

**坡印廷向量防御**：常规防御屏蔽的是来自 xyz 方向的能流，对 $w$ 轴方向的能流无效。

**几何原理**：
$$\text{攻击路径} = (x_t, y_t, w_0) \to (x_t, y_t, w_t)$$

投射物保持目标的 $x$-$y$ 坐标不变，在 $w$ 轴上滑动，从目标的 $w$ 切面"落下"。

### 4.2 需要知道的信息

**必须获取目标的精确 $(x, y, w)$ 坐标**。

这是设计的关键约束：**攻击即侦察，侦察即暴露**。

- 对目标进行感知扫描：获取坐标，但你的位置被目标探测到
- 跨维度攻击发射：暴露你的 $w$ 值给目标，但攻击路径难以防御

关联：[[1.0 炮弹动力学]]

### 4.3 伤害计算

攻击穿越 $\Delta w$ 时，通过所有中间 $w$ 层，对每层产生**衰减伤害**：

$$D_k = D_0 \cdot e^{-\alpha \cdot |w_k - w_0|}$$

其中：
- $D_0$：基础伤害
- $\alpha$：衰减系数（由材料"w轴导电率"决定）
- $w_k$：第 $k$ 层的 $w$ 值

**战术含义**：攻击路径经过的层越多，附带伤害越多。计划攻击路径成为战术考量。

## 5. $m_w$ 弹簧系统（Godot 实装规格）

### 5.1 概念

每个游戏对象都有一个**目标 w 值** $w_{\text{target}}$（它"本来应该在"的层）。

当 Wanderer 拖拽它到新的 $w$ 值时，弹簧力试图把它拉回原位：

$$F_w = -k_{m_w} \cdot (w_{\text{current}} - w_{\text{target}})$$

其中 $k_{m_w}$ = 该对象的 $m_w$ 值（弹簧刚度）。

### 5.2 分类参数表

| 对象类型 | $m_w$ | 感知跟随速度 | 视觉 |
|---------|-------|------------|------|
| 岩石、建筑 | $m_w = \infty$ | 不跟随 | Wanderer 穿越时直接淡出 |
| 普通 NPC | $m_w = 50$ | ~10 秒滞后 | 慢慢淡出，方向固定 |
| 同伴（初始）| $m_w = 10$ | ~3 秒滞后 | 轻微模糊，追着跟 |
| 同伴（熟悉后）| $m_w = 3$ | ~0.5 秒 | 几乎同步 |
| 节点构造体 | $m_w = 0.1$ | 立即同步 | 已"适应"w轴 |
| Wanderer | $m_w \approx 0$ | — | 基准参照 |

### 5.3 Godot 实现思路

```gdscript
# 每个对象挂载的 WAxisComponent
class_name WAxisComponent
extends Node

@export var m_w: float = 10.0        # w 轴惯性质量
@export var w_target: float = 0.0    # 本体所在的 w 层

var w_current: float = 0.0

func _physics_process(delta: float) -> void:
    var wanderer_w = GameState.wanderer_w
    
    # 弹簧拉力（向目标 w 层回弹）
    var spring_force = -m_w * (w_current - w_target)
    
    # Wanderer 的感知场拖拽力
    var drag_force = _calculate_drag(wanderer_w)
    
    # 积分更新（简单欧拉，后期换 Verlet）
    var w_velocity = (spring_force + drag_force) * delta
    w_current += w_velocity
    
    # 根据 w_current 与 wanderer_w 的差值更新透明度
    _update_visibility()

func _calculate_drag(wanderer_w: float) -> float:
    var delta_w = wanderer_w - w_current
    # 感知场衰减：距离越远，拖拽力越弱
    var falloff = exp(-abs(delta_w) * 2.0)
    return delta_w * falloff * (1.0 / max(m_w, 0.01))

func _update_visibility() -> void:
    var wanderer_w = GameState.wanderer_w
    var delta_w = abs(w_current - wanderer_w)
    var alpha = 1.0 - smoothstep(0.2, 0.8, delta_w)
    # 设置所属节点的透明度
    get_parent().modulate.a = alpha
```

## 6. 能流瓶（资源系统）

**Gemini 建议**：W 轴移动消耗"能流瓶"，使 $w$ 平移成为战术资源。

**设计决定**：不直接限制移动，而是：

| 行为 | 资源消耗 |
|------|---------|
| 自由 w 移动（$|\Delta w| < 0.5$）| 无消耗 |
| 快速大幅跳跃（$|\Delta w| > 0.5$）| 消耗"能流储备" |
| 跨维度攻击 | 消耗能流储备 |
| 感知扫描 | 小量消耗，但主要代价是暴露位置 |

能流储备上限：10 格  
回复方式：自然恢复（慢）、吸收节点能量（快）、诺埃尔的补给物品

## 7. 指示器噪声：晶格牧师反射器的干扰模型

反射器向真实信号方向注入一个**相位偏移的伪信号**，方向差固定（Gemini 提到萨奇发现的 17 度偏移）：

$$\vec{v}_{\text{显示}} = \vec{v}_{\text{真实}} + \alpha_{\text{reflector}} \cdot \vec{v}_{\text{fake}}$$

萨奇加入后，她提供修正向量：

$$\vec{v}_{\text{校正}} = \vec{v}_{\text{显示}} - \vec{v}_{\text{fake,known}}$$

不是完美消除，是**逐步逼近**——反射器密度越高，校正越难。

## 8. 称号 × 感知 × 战斗 三轴进阶表

| 称号 | 来源 | 主要效果 | 深层加成 |
|------|------|---------|---------|
| 战士 | 近战击败敌人 | 攻击力 +，防御 + | 跨维度攻击基础伤害 + |
| 先知 | 感知扫描次数 | $\Delta w$ + ，噪声识别率 + | 可感知 Princess 的情绪层 |
| 学者 | 阅读遗迹文献 | 解读古文字、配方 | 部分对话新选项解锁 |
| 旅者 | NPC 对话 | 商店折扣，信息量 + | 某些 NPC 隐藏线触发 |
| 长丝 | 在极深层长时间停留 | $m_w$ 进一步降低 | w 轴移动不消耗能流 |

**"长丝"称号**：是 Wanderer 开始理解自己的超维长丝本质的象征。游戏后期的独特称号，影响结局 B 的谜题难度。
