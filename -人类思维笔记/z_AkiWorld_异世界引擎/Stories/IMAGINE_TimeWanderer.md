# SIDE A — 浪漫层：世界观与主旨

![[PIC_timewanderer_worldbuilding_crystalline_spacetime_lattice_fourth_axis_dormant_amber_light.png]]

> *"四个维度本应同等地铺展。  
> 但在某一刻，三个维度学会了——或者说，被迫学会了——把第四个遗忘。"*

---

## 宇宙学前提：被遗忘的轴

在 Time Wanderer 的宇宙里，时间**不是**第四维度。

空间有四条轴：$x, y, z$ 和 $w$。时间是另一件事，它流动，但它不是空间。

那么 $w$ 轴是什么？

$w$ 轴是空间的**深度方向**——不是高低，不是远近，而是一种人类感官从未进化出器官去感知的方向。就像一张纸上的生命感知不到"厚度"方向，三维生命也感知不到 $w$ 方向。

但它就在那里。一直都在。

---

## 时空晶体结构

古代文献（以晶格符文的形式残存于遗迹维度）记录了一个事实：

**宇宙的时空结构是晶体的。**

不是比喻——是字面意义上的晶体。时空本身是一张格点网络，每个格点是一个 $(x, y, z, w)$ 坐标。晶格有各向异性：

$$\sigma_x = \sigma_y = \sigma_z \gg \sigma_w$$

在 $xyz$ 方向上，力、信息、生命的传播阻力极小。在 $w$ 方向上，阻力极大——不是无穷大，只是"大到几乎没有任何自然力会愿意走 $w$ 方向"。

用一个不完全准确但有诗意的类比：$w$ 方向的时空传导率，就像琥珀里的电阻。物质会在 $xyz$ 里流动，但会被"晶格键"锁在某个 $w$ 层上，像被嵌在了琥珀深处。

这就是为什么主事件永远发生在三个维度里：**不是不能去第四维，是代价太高。**

> 物理学精确描述：在 $w$ 方向施加力 $F_w$，对应的位移产生极大的"时空坡印廷矢量"——能量从电磁场流入这个方向会迅速耗散为某种低温背景辐射。
> 
> 通俗描述：向 $w$ 方向移动，就像在粘稠的蜂蜜里游泳，同时蜂蜜还在把你的能量喝掉。

---

## 遗迹维度

$w \neq 0$ 的世界不是空的。

它们是**遗迹**。

在宇宙的某个更早的时代，$w$ 轴的传导率并不总是这么低。曾经有过一段时期，文明在四维空间里自由流动，建造了跨越 $w$ 层的结构——建筑的地基在 $w=0$，但顶部延伸到 $w=7$；道路在 $xyz$ 平面展开，但在 $w=3$ 有一条只有特定人才能走的密道。

然后发生了什么。晶格键**突然强化**了——在某一天，某一刻，不知道为什么，$\sigma_w$ 骤降至现在的值。

四维的文明在瞬间被冻结在了各自的 $w$ 层上。  
他们的建筑、他们的道路、他们的遗体——都还在那里。  
被锁在不同的 $w$ 层，成为遗迹。

**这是世界观的第一个秘密：遗迹维度不是废土，它是博物馆。每一个 $w$ 层，是那场灾变时某个地方的冰冻截面。**

---

## Wanderer 与 Princess

**Wanderer** 是一个异常。

不知道为什么——基因突变、辐射事故、或者某种更玄妙的宿命——他的 $m_w$（w 方向的惯性质量）极低。几乎为零。

这意味着他可以以极低的能量代价在 $w$ 轴方向移动。在主世界（$w=0$）的人眼里，他会"消失"——实际上只是滑进了 $w=0.1$ 的层。再滑一点，$w=1$，$w=2$，进入遗迹维度的深处。

他这辈子大部分时间都活在主世界，假装自己是普通人。

直到他遇到了 Princess 留下的信号。

---

**Princess** 不是公主的字面意义。

"Princess" 是 Wanderer 给她起的名字，因为他第一次感知到她的存在时，只是遗迹维度深处某个 $w$ 层传来的一束微弱的信号——像是有人在黑暗里点了一根蜡烛。

她不在 $w=0$。她从来不在 $w=0$。

她存在于某个比他能轻易到达的 $w$ 层更深的地方，每隔一段时间发出一次信号，然后漂移——像被 $w$ 轴上的洋流推着走，或者像有意在躲避什么。

Wanderer 不知道她是谁，不知道她的 $xyz$ 在哪里，不知道她在 $w$ 轴的哪个深度。

他只知道信号的方向。他只会追。

**这是游戏的叙事核心：一个追踪行为，在四维空间里展开。**

---

## 叙事主旨

> *三维生命的眼睛，只能看到四维存在的投影。*  
> *但爱从不局限于维度——*  
> *它是第一个学会穿越 $w$ 轴的力。*

Time Wanderer 讲的不是爱情故事。它讲的是**感知的成长**。

玩家一开始只能平移 $w$ 轴，感知 Princess 的距离和方向。这就像在二维平面上，用一根杆子戳进第三维，判断里面有没有东西。

随着游戏进展，玩家解锁对 $w$ 空间的更精细感知：从"距离"到"形状"，从"在哪里"到"是什么"，从"信号"到"对话"。

**最后一关不是击败Boss。是 Wanderer 第一次在四维空间里真正"看到"Princess——不是投影，不是切片，而是完整的四维存在。**

对于一个只有三维感知的生命来说，那一刻的体验，没有任何三维语言可以描述。

游戏在那一刻结束。不给答案，只给体验。

## 降维 MVP 约定

**2D MVP 的坐标系**：

| 概念 | 实际坐标 | Godot 实现 |
|---|---|---|
| "3D 世界" | $(x, y)$ 平面 | 2D 场景 |
| $w$ 轴 | 独立标量 $w \in \mathbb{R}$ | 全局变量 / World State |
| "4D 位置" | $(x, y, w)$ | `Vector2` + `float` |
| w 层切片渲染 | 在 $w = w_{\text{player}}$ 处渲染主世界 | Shader uniform |

**约定**：Wanderer 的 $w_{\text{player}}$ 是**连续实数**，不是离散层。离散层只是渲染上的简化，底层始终是连续的。

**2D MVP 中 $w$ 维等价关系**：

```
真实4D游戏          2D MVP等价
(x, y, z, w)  →   (x, y, w)
z轴             →   省略（玩家始终在z=0平面）
w轴的切片预览   →   背景层的透明度/色调变化
```

---

## 核心状态机

Wanderer 在任何时刻处于以下状态之一：

```
States:
┌─────────────────────────────────────────────────────┐
│  ANCHORED          │ w_player 固定，正常移动xyz      │
│  DRIFTING          │ w_player 正在平移，xyz被限制    │
│  PHASED            │ w_player ≠ 0，在遗迹维度中     │
│  LOCKED            │ 遭遇高密度晶格区，w移动暂时封锁 │
│  RESONATING        │ 在Princess的信号附近，感知增强  │
└─────────────────────────────────────────────────────┘

Transitions:
ANCHORED  →  DRIFTING     触发：玩家按下 w 轴移动键
DRIFTING  →  ANCHORED     触发：w 平移完成，松开键
ANCHORED  →  PHASED       触发：w_player 超出阈值 δ（默认 δ=0.5）
PHASED    →  RESONATING   触发：|w_player - w_princess| < ε_sense
任意       →  LOCKED       触发：进入高 σ_w 晶格区
```

## Indicator 数学

Wanderer 拥有一个"维度感知器"，显示两个分量：

### 分量 1：w 轴距离

$$d_w = w_{\text{princess}} - w_{\text{player}}$$

**显示方式**：纵向条形，中心为0，向上/下伸展表示 Princess 在 $+w$ 或 $-w$ 方向，长度线性映射到 $|d_w|$。

**噪声**：感知器不是精确的。测量值加入 Perlin 噪声：

$$\tilde{d}_w = d_w + \eta \cdot \text{Perlin}(t, w_{\text{player}})$$

噪声幅度 $\eta$ 随 $|d_w|$ 增大——越远，越模糊。

### 分量 2：xyz 方位角

$$\vec{v}_{xy} = \frac{(x_p - x_w, \ y_p - y_w)}{|(x_p - x_w, \ y_p - y_w)|}$$

**显示方式**：圆形罗盘，指针指向 Princess 在主世界平面的**投影方向**。注意这是 $(x,y)$ 的方向，与 $w$ 无关。

**距离叠加**：方位角圆的半径大小反映 $xy$ 平面距离：

$$d_{xy} = \sqrt{(x_p-x_w)^2 + (y_p-y_w)^2}$$

越远，圆越大（或越模糊）。越近，圆越清晰、越小。

### 综合距离（用于难度设计）

$$d_{\text{total}} = \sqrt{d_{xy}^2 + \lambda(w) \cdot d_w^2}$$

其中 $\lambda(w)$ 是 $w$ 感知权重，随 Wanderer 的能力成长而增大。初始 $\lambda \approx 0.1$（几乎感知不到 $w$ 差距），后期 $\lambda \to 1$（$xyz$ 和 $w$ 方向等权）。

**这个成长曲线就是玩家的认知曲线。**

## 跨维攻击几何

> 核心问题：我在 $(x_0, y_0, w_0)$，敌人在 $(x_e, y_e, w_e)$。我要从 $w_0$ 的一侧打出一个抛体，让它在经过 $w_e$ 时，恰好经过 $(x_e, y_e)$。

**抛体轨迹参数化**：

$$P(t) = (x_0 + v_x t,\ y_0 + v_y t,\ w_0 + v_w t)$$

**命中条件**：

$$\begin{cases} w_0 + v_w \cdot t^* = w_e \\ x_0 + v_x \cdot t^* = x_e \\ y_0 + v_y \cdot t^* = y_e \end{cases}$$

由第一式解出命中时间 $t^* = \dfrac{w_e - w_0}{v_w}$，代入：

$$v_x = \frac{x_e - x_0}{t^*} = \frac{(x_e - x_0) \cdot v_w}{w_e - w_0}, \quad v_y = \frac{(y_e - y_0) \cdot v_w}{w_e - w_0}$$

**结论**：只要知道敌人的完整 $(x_e, y_e, w_e)$，就能解出唯一的发射方向向量 $(v_x : v_y : v_w)$（方向比例）。$v_w$ 的大小决定抛体速度。

**玩法含义**：

```
不知道 w_e → 无法瞄准 → 必须先侦察敌人的 w 层
知道 w_e 不知道 (x_e, y_e) → 可以打"w层扫射"，面积杀伤
知道完整位置 → 精确制导，最强
```

攻击即侦察。侦察深度决定攻击精度。这是一个**信息驱动的战斗系统**。

### 跨层残影伤害

抛体穿越中间 $w$ 层时产生"余震"——在经过的每个 $w$ 层的 $(x(t), y(t))$ 位置造成衰减伤害：

$$\text{Damage}(w_{\text{mid}}) = D_0 \cdot e^{-\alpha |w_{\text{mid}} - w_e|}$$

这意味着一次精准的跨维发射，可以同时打击多个 $w$ 层上处于相近 $xy$ 位置的敌人。但代价是：**暴露了你的 $xy$ 位置给所有那些层。**

## w 质量系统

每个物体拥有一个 $m_w$ 属性，决定它在 $w$ 轴方向的行为：

| $m_w$ 范围 | 类型 | 行为 | 游戏对象示例 |
|---|---|---|---|
| $m_w = \infty$ | **w-锚定** | $w$ 坐标永不改变 | 地形、古代建筑、晶格节点 |
| $m_w \gg 1$ | **w-惰性** | 可以被推入 $w$ 轴，但极慢 | 大型机械、精英敌人 |
| $m_w \sim 1$ | **w-弹性** | Wanderer 走过时，以弹簧模型跟随 | 普通敌人、物品 |
| $m_w \ll 1$ | **w-漂移** | 几乎自由漂移，不稳定 | 灵体、信号碎片 |
| $m_w \approx 0$ | **w-自由** | Wanderer 本人 | — |

**弹簧追踪模型**（$m_w \sim 1$ 的物体）：

$$\ddot{w}_{\text{obj}} = -k(w_{\text{obj}} - w_{\text{player}}) - c\dot{w}_{\text{obj}}$$

$k$ = 弹簧刚度（质量越低越松软），$c$ = 阻尼（防止震荡）。

**视觉效果**：
- $w$-锚定的地形：Wanderer 切换 $w$ 层时，这些物体从屏幕上淡出（透明度衰减 + 色调偏移）
- $w$-弹性的敌人：被拖着走，但有延迟尾迹，看起来像"影子"
- $w$-漂移的灵体：在 $w$ 轴上随机游走，构成 Seek game 的移动目标

## 关卡结构

### Seek 关卡的数学骨架

每个 Seek 关卡定义如下：

```python
class SeekLevel:
    # 地图：2D 平面 (x, y) 上的 SDF 障碍场
    obstacle_field: SDF2D
    
    # w 轴范围
    w_range: tuple[float, float]  # e.g. (0, 10)
    
    # Princess 的运动轨迹（参数化）
    princess_trajectory: Callable[[float], tuple[float, float, float]]
    # 返回 (x, y, w) 作为时间 t 的函数
    
    # 晶格阻力场：某些区域 w 移动被减速
    w_resistance_field: Callable[[float, float], float]
    # 输入 (x, y)，返回局部 σ_w 值
    
    # 感知噪声等级
    perception_noise: float  # 0=完美感知, 1=几乎全噪声
```

### Princess 的逃逸策略

Princess 的运动不是随机的——她有策略：

```
if perceived(wanderer_position):
    # 感知到 Wanderer 时：
    # 在 w 轴上加速漂移，方向远离 Wanderer 的 w 层
    Δw = -sign(w_player - w_princess) * escape_speed
    # 在 xy 平面上向"w 阻力大"的区域躲避（那里 Wanderer 难以追踪）
    target_xy = argmax(w_resistance_field)
else:
    # 未被感知时：缓慢漂移，偶尔发出信号
    Δw = drift_noise(t)
    emit_signal()
```

**感知-暴露机制的完整闭环**：Wanderer 感知 Princess → Princess 逃跑 → Wanderer 追 → Wanderer 暴露自己的位置 → 遗迹维度的守卫也感知到 Wanderer → 守卫追击。

## 感知-暴露悖论

这是整个玩法系统最核心的抽象机：

```
感知行为    →   释放 w 轴"涟漪"（类电磁辐射）
涟漪强度   ∝   感知精度
涟漪传播   →   被感知到的所有实体都能定位你

结论：
越精确地感知对方 → 越精确地暴露自己
```

数学模型：

$$\text{Signal}_{\text{emitted}}(t) = \nabla_{(x,y,w)} \cdot \text{PerceptionField}(t)$$

即你的感知场的散度，等于你向外辐射的信号强度。**感知本身是一种波**，而波是双向的。

**玩法设计含义**：

| 玩家行为 | 感知精度 | 暴露程度 |
|---|---|---|
| 静止不感知 | 0 | 0 |
| 被动监听（低功率）| 低，但持续 | 极低 |
| 主动扫描（高功率）| 高，瞬时 | 高，瞬时爆发 |
| 发射跨维攻击 | 已知目标位置 | 最高（攻击路径完全暴露） |

Princess 会学习你的感知节奏。如果你每隔固定时间扫描一次，她会在两次扫描之间移动，精确避开你的下一次扫描。这把它变成了一个**频率博弈**。

# 交汇层：两种风格的共同根

> 世界观说："$w$ 轴是被遗忘的空间，遗迹在那里沉睡。"  
> 玩法说："$w$ 轴是信息战场，感知与暴露的博弈在那里展开。"

两者根是同一棵树：

**在一个高阻力的维度里，感知本身就是最稀缺的资源。**

遗迹维度之所以是遗迹，是因为信息难以在 $w$ 方向流动。那里的一切都是"信息孤岛"——被锁在自己的 $w$ 层，无法与相邻层沟通。Wanderer 的低 $m_w$ 不只是一个位移能力，它是一个**信息桥接能力**：他能在孤岛之间移动，这让他在世界观里是传说，在玩法里是最强的侦察者。

Princess 在 $w$ 轴上漂移，是因为她也是一个 $m_w$ 极低的存在——但她是主动利用 $w$ 轴来**保持孤立**。她在所有层之间移动，正是为了让任何人都无法固定她的位置。她是信息论意义上的**最大熵存在**。

追踪她，就是在和熵作斗争。

> *"她住在所有可能性里。  
> 我住在所有可能性的边上，努力缩小范围。  
> 这就是全部故事了。"*  
> — Wanderer 的感知日志，$w=3.7$，日期不明
