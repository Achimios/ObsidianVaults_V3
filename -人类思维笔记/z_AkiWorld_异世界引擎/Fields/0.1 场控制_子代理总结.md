# AkiWorld 场控制系统 — 设计总结

## 核心哲学

最少控制点 × 最高可控度 × 艺术感——三者缺一不可。

不能像往水里丢石头那样让玩家目睹一切然后无能为力，也不能把战场做成棋盘让每一步都可预测。流场控制的设计目标就是维持这两端之间的张力：玩家感觉自己在驾驭一匹野马，而不是在推一块砖头。

毛球定理不是目的地，是引子。真正的目的是：理解封闭表面的奇点是必然存在的，然后主动把这个"必然"变成设计工具。

---

## 关键设计决策

### 1. 为何选离散向量场（差分迭代，不用微分）

游戏跑 120fps。微分方程在这里是奢侈品——精度没人在意，连续解也没人买单。

离散向量场 + 差分迭代是正确选择：
- 没有复杂碰撞体系，计算量本身就极低
- 差分迭代天然适配帧循环，每帧推进一步，逻辑清晰
- 精度"足够好"即可，游戏不是物理论文
- 保留了流体的高动态感，同时消耗极低

**结论**：不要因为"物理正确"而引入微分求解器，差分迭代是这个场景的最优解。

---

### 2. 奇点作为控制杠杆

奇点是流场里信息密度最高的点，也是玩家唯一需要操心的东西。

三种基础奇点各有"游戏性格"：
- **源点（Source）**：爆炸核心，把周围一切往外推。用于清场、打散敌群。
- **汇点（Sink）**：漩涡，把周围一切往内吸。用于聚集、诱捕、引导。
- **鞍点（Saddle）**：十字路口，沿两轴分流。用于分割战线、制造夹角通道。

理想控制粒度：**3～7 个可控奇点**，多了变混沌，少了变无聊。

玩家不直接攻击敌人，而是通过移动、增强、削弱奇点，让流场替玩家"说话"——把敌人引入伴生焦点，然后利用拓扑约束让对方无路可走。这是贝叶斯推演驱动的间接控制，而不是 WASD 硬操。

---

### 3. 2D + 投影的实现策略

只在二维平面上做 **Simulation**，高度方向全部是 **Emulation**。

分层原则：
- **面上**：完整的离散向量场模拟，奇点计算，流线更新，粒子推进
- **高度**：视觉层，仅做视差和投影变形，不参与物理计算
- **非欧几何地图**（地形不平）：投影适配（Projection），不需要重写场方程
- **函数不连续**（地形边界、奇点附近）：Phong 平滑处理，保证视觉连续性

这个分层让"看起来三维"和"跑得飞快"同时成立。甚至可以扩展到三阶向量场（向量场的向量场），但优先跑通二维原型再说。

---

### 4. 可玩性边界

这是设计的核心张力，必须主动维持：

| 太混沌 | 最优区间 | 太确定 |
|--------|----------|--------|
| 往水里丢石头 | 玩家感觉"我在控制野马" | 棋盘 |
| 奇点太多/太强 | 3～7 个奇点，局部扭曲 | 奇点太少/太弱 |
| 敌人被随机扫走 | 可预判但有戏剧性 | 敌人轨迹完全可算 |

守门机制：基础流场保留全局趋势（整体旋转/漂移），奇点只做局部扰动。敌人 AI 不傻也不神，留出"观察 → 微调 → 看到戏剧性结果"的反馈循环。

---

### 5. 艺术-机制统一原则

**The Glitch Mob 的音乐节拍 = 向量场的脉动节奏。**

战斗的图案不是随机的，是流线的拓扑结构在时间轴上的展开。

实操含义：
- 场强度/奇点强度随音乐节拍变化（beat → pulse）
- 流线可视化本身就是战斗的"乐谱"——玩家既是演奏者也是指挥家
- 视觉语言（粒子轨迹、色彩扭曲、空间变形）必须和音乐节奏对齐，而不是独立装饰

**音乐是运动的节奏，战斗的图案是流线拓扑。** 这两件事是同一件事。

---

## 毛球定理的实际用途（简短）

不用去证明它，只需要记住一句话：**封闭表面上必然存在奇点。**

设计含义：与其让奇点随机产生（bug 感），不如主动设计奇点的位置、数量和性格，让它成为玩法的核心资源。毛球定理是"大自然替你在地图上插旗子"，设计师要做的是决定旗子插在哪里、插几根。

## 下一步设计任务

1. **二维离散向量场原型**：在网格上实现源/汇/鞍三种奇点，用粒子可视化流线，验证"3 个奇点能控制多复杂的战场"。
2. **音乐同步机制**：接入 The Glitch Mob 节拍数据，让奇点强度随 beat 脉动，测试艺术感与可玩性的融合点。
3. **奇点操控界面设计**：确定玩家以何种方式控制奇点（拖拽/技能释放/参数映射），目标是"感觉像在拨弄时空"而不是"感觉像在调参数"。

## 技术实现提示

**离散向量场数据结构**：
- 二维网格，每个格点存一个 `(vx, vy)` 向量
- 更新规则：每帧差分迭代，叠加各奇点贡献（距离衰减的向量叠加）
- 可选：稀疏格点 + 双线性插值，平衡精度与开销

**奇点数据表示**：
```python
Singularity = {
    "type": "source" | "sink" | "saddle",
    "pos": (x, y),          # 网格坐标
    "strength": float,       # 正/负控制方向
    "radius": float,         # 影响半径
}
```

**流线可视化方向**：
- 粒子系统：每帧按当前格点向量推进，生命周期短（残影感）
- 色彩映射：速度大小 → 亮度/饱和度，奇点附近 → 颜色突变
- Phong 平滑：奇点边界处对向量做插值，避免视觉跳变
- 音乐节拍触发：beat 时刻向所有粒子施加一次脉冲扰动
