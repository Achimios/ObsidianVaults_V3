## 一、共振频率追踪：SDFT → Goertzel 两阶段流水线

### 1.1 BetaFlight现有方案的局限

BF动态陷波（动态Notch）：基于**72点SDFT**（滑动离散傅里叶，`common/sdft.h`），得到36个频率bin（bin 0为DC弃用，有效bin 1~35）。

源码参考：`flight/dyn_notch_filter.c#L130` — 共振峰通过SDFT频谱寻极大值定位。

**问题：** 飞行中电机转速变化快，共振窗口可能只持续20~50ms。72点@2kHz = 36ms采集时间，窗口内才能定位——但共振可能刚出现就已经离开了共振频率。

### 1.2 改进方案：SDFT诊断 + Goertzel实时跟踪 + 半静态记忆

```
阶段1 (诊断层，滑动SDFT)
  → 发现频谱峰值：f_resonance_1, f_resonance_2 (最多2~3个)
         ↓
阶段2 (跟踪层，Goertzel)
  → 只盯住已知频率点，O(N_target)代价极低
  → 实时监测这些频点的能量，决定陷波深度
         ↓
半静态记忆
  → 将f_resonance_1, f_resonance_2写入Flash
  → 下次上电直接激活，跳过诊断期
  → 若下次飞行该频率点能量异常低 → 修正记忆（换了桨/电池改变了刚度）
```

**Goertzel算法核心（单频点DFT）：**

$$y[n] = x[n] + 2\cos(2\pi k/N) \cdot y[n-1] - y[n-2]$$

计算代价仅为 $O(2N)$ 加法，远低于完整FFT的 $O(N\log N)$，适合每帧实时跑2~4个频点。

```c
typedef struct {
    float coeff;      // 2*cos(2π*k/N)
    float s1, s2;     // 递推状态
    int   n;          // 当前采样计数
    int   N;          // 窗口长度
} GoertzelState;

float goertzel_process(GoertzelState *g, float x)
{
    float s0 = x + g->coeff * g->s1 - g->s2;
    g->s2 = g->s1;
    g->s1 = s0;
    g->n++;
    if (g->n >= g->N) {
        // 完整窗口，计算能量
        float real = g->s1 - g->s2 * cosf(M_PI * (float)(g->n - 1) / g->N); // 简化
        float imag = g->s2 * sinf(M_PI * (float)(g->n - 1) / g->N);
        g->n = 0; g->s1 = g->s2 = 0;
        return real * real + imag * imag;
    }
    return -1.0f;  // 未满窗口
}
```

## 二、前馈（FF）的本质与代价

### 2.1 FF的物理作用

**FF（Feedforward）**：不是基于误差，而是直接基于setpoint导数（角速度变化率）计算控制量。它绕过整个反馈回路，强制在误差还没产生时就提前给油。

$$u_{FF} = K_{FF} \cdot \frac{d(\text{setpoint})}{dt}$$

**实测数据（2207电机 + 6042×3桨）：**
- 电机响应时间：**7ms**（油门50%阶跃到100%，到达90%RPM的时间）
- 无FF时阶跃响应：**18ms**（从打杆到实际角速度达到目标的90%）
- 有FF时阶跃响应：**10ms**（FF强制提前给油，减少了约8ms感知延迟）
- **代价：** FF必然带来过冲（约15~25%），需要D项抑制

### 2.2 FF与D的联合使用规则

| 场景 | FF | D |
|------|----|----|
| 追求极限响应速度 | 大（可适当过冲）| 需要同步增大阻尼 |
| 追求稳定/鲁棒 | 小或关 | 正常D |
| 机体刚性高 | 可用大FF | D可偏小 |
| 机体柔性/振荡倾向 | 小FF | D要保守 |

**PD比例的建议获取方法**：先用低I + 低FF的设置获得纯PD比例，再单独调整I（消日常静差）和FF（调响应速度）。

### 2.3 洗桨（Prop Wash）时的参数策略

洗桨 = 快速极低油门→大油门瞬间，4个旋翼的气动干扰区建立有延迟，扰动是**低频脉冲性质**。

| 参数 | 洗桨时调整方向 | 原因 |
|------|---------------|------|
| P | 增大（+20~40%）| 低频扰动靠P抵抗 |
| I | 适当增大 | 消除洗桨导致的积累偏差 |
| D | 减小（-30~50%）| 洗桨时震动大，D会放大高频，容易发散 |
| 低通滤波截止频率 | 提高（减少滤波）| 洗桨需要快速响应，低通延迟此时是敌人 |
| TPA | 旧版TPA削减PID其实此时有副作用 | 新版TPA只削减D更合理 |

> **为什么新版TPA只削减D？**  
> 大油门下震动幅度大，D作用更强烈，高D容易发散。  
> 同时大油门下桨惯性/气动阻力/气流扰动削减了P的作用，所以相对提高PD比是合理的。

## 三、Yaw Bounce 与 Yaw Jump 的机械补偿

### 3.1 物理根因

低油门悬停时快速打Yaw：一对对角电机降至怠速，另一对加油门。  
当Yaw PID（特别是回弹回中时的反向Yaw PID）使得：

$$\frac{\text{加速电机油门}}{2} \gg \text{设定悬停油门}$$

快速打Yaw，→ 机体短时升高（Yaw Jump）→ 回弹时再次更猛升高（Yaw Bounce）

### 3.2 补偿逻辑

**在快速Yaw打杆时：**
1. 积分计算4电机平均油门比setpoint油门多出多少（$\Delta\bar{T} = \bar{T}_{actual} - T_{hover}$）
2. 积分持续时间 $t_{excess}$
3. 在Yaw角减速阶段（摇杆回弹回中），延迟地施加一个等量的向下油门补偿：

$$\delta T_{compensate} = -k \cdot \Delta\bar{T} \cdot t_{excess}$$

**在回弹回中后的Yaw Bounce：**
- 快速降低油门一个小脉冲，把之前高度抖动量延迟地补偿回去

**注意**：这是基于BF固件层面的修改，适合自研飞控，不适合所有情况。

## 四、AHRS质量提升：动态权重 + 碰撞处理

### 4.1 Madgewick / Mahony 动态Kp/Ki

标准Madgewick/Mahony用固定的加速度计权重（β参数）。  
改进：**动态调整加速度计比重**：

```c
float imu_get_acc_trust(float acc_mag, float gyro_rate)
{
    // 当前加速度模长接近重力 → 可信
    float g_deviation = fabsf(acc_mag - 9.81f) / 9.81f;  // 0=完美重力
    // 角速度小 → 加速度计更可信
    float gyro_weight = 1.0f / (1.0f + gyro_rate * 0.02f);
    // 综合可信度
    float trust = (1.0f - g_deviation * 2.0f) * gyro_weight;
    return fmaxf(0.0f, fminf(1.0f, trust));
}
```

**理由**：高角速度时加速度计被离心力污染；加速度模长=g表明当前只有重力，加速度计准确。

### 4.2 碰撞检测与电机保护

```c
void imu_collision_handler(float acc_mag, float gyro_rate)
{
    static float collision_timer = 0;
    
    // 碰撞判断：加速度瞬间超量程或陀螺仪极大
    if (acc_mag > 50.0f || gyro_rate > 2000.0f) {  // deg/s
        collision_timer = COLLISION_FREEZE_TIME;    // 暂时短时提高加速度计权重
        aki_set_acc_kp(HIGH_KP_COLLISION);
    }
    
    // AirMode处理：碰撞时短时关闭（0油门不PID，防止窜跳）
    if (collision_timer > 0) {
        aki_disable_airmode();
        collision_timer -= dt;
    } else {
        aki_restore_airmode();
    }
    
    // 电机堵转检测 → 闭锁
    if (motor_stall_detected() && stall_time > STALL_LOCK_THRESH) {
        aki_disarm();  // 需要手动重新解锁
    }
}
```

### 4.3 陷在墙面的处理（角度约束）

## 五、多电机混用问题

### 5.1 不同KV电机 / 不同桨叶混用

- **最大升力** = 最弱桨叶的升力×4（水桶效应）
- **最大转速** = 最低KV电机的最大转速限制其余电机
- **4电机均值油门**并非总等于设定油门，不同KV时最优均值方法不同

**建议**：始终使用匹配的相同规格4个电机+桨叶。若必须混用，在Mixer层加入静态补偿系数。

### 5.2 油门物理本质（原草稿精华，已整合）

$$V_{eff} = V_{BAT} \times \text{duty\_cycle}$$

低电量（VBAT↓）时，相同油门产生的推力减小，需要提升PID增益补偿。  
⚠️ 原`推力线性化`作用于油门(然后自然作用于PID)。应该单独作用于PID，否则飞手无法感知低电量，且80%RcCommand时可能SetPoint就100了%影响手感线性度。

## 六、PID自动调参的收敛策略

### 原草稿思路（验证后保留）

> PID的本质是对 油门电压、电机响应时间、桨叶升力、滤波相移、机体惯量和阻力 的综合迭代优化。  
> 非线性因素存在，不存在精确的传递函数拟合。可以简单线性迭代。

**推荐收敛流程（对应AkiDynamics中的四档参数体系）：**

```
Step 1: 用低I + 低FF → 获得纯PD响应曲线
Step 2: 用超调量→ζ公式推算当前ζ
Step 3: 调P到"临界振荡前的80%"
Step 4: 调D到刚好阻尼超调但不引入高频
Step 5: 加入最小I消除静差
Step 6: 加FF到目标阶跃响应时间（接受约15~25%过冲）
Step 7: 验证TPA: 高油门下重复Step 1~6
Step 8: 固化四档参数，动态切换
```

**发散时的立即响应：**
1. 检测到峰度>7 **或** D项方差>阈值 → **立即**：降P至当前×0.4，升主LPF至保守值，激活陷波
2. 不要等振荡完全稳定后才降，预判比事后反应重要10倍
