## 一、完整六层分层（修订版，含与1.0的衔接）

| 层 | 硬件 | 频率 | 职责 |
|---|---|---|---|
| **云端 LLM/VLM** | 云服务器 | 0.1–0.3Hz | 飞行任务理解、长期规划、场景语义分析 |
| **本地 PID 分析模型** | 本地PC（离线） | 离线 | 黑匣子整定 → 生成 PID 参数表 + 增益调度脚本（→ 1.0详述） |
| **本地意图小模型** | 地面站 | **1Hz** | 宏观PID策略选择（激进/保守/抗风模式）、路径规划、战术决策；通过数传下发"模式指令" |
| **机载电脑** | 树莓派 / Jetson | **50Hz** | 实时工况感知（洗桨/重载/小桨/强风）、在地面站策略框架内微调增益（→ 1.0详述） |
| **STM32 动态层** | 飞控 | 200Hz–2kHz | 增益调度执行，TPA / D-term / I-relax（→ 1.0详述） |
| **STM32 内环** | 飞控 | 2kHz | PID 硬实时闭环，不依赖上层（→ 1.0详述） |

### 关键区分：地面站 vs 机载

| | 本地意图小模型（地面站） | 机载树莓派（50Hz） |
|---|---|---|
| **知道什么** | 飞行任务目标、地图、预期环境 | 当前实时传感器数据 |
| **决策粒度** | 粗（选策略包） | 细（微调增益值） |
| **例子** | "要飞穿树林 → 选保守模式" | "检测到洗桨 → 压D项20%" |
| **输出** | 策略编号 + 参数上下限 | 具体ΔKp/ΔKd/ΔTPA |
| **依赖** | 任务规划、LLM摘要 | VMD特征、NN分类器 |

> 结论：两者是上下级关系，**不竞争**。地面站给框架，机载在框架内响应。
