## 一、AkiRouter四层架构（正式版）

```
┌──────────────────────────────────────────────────────┐
│  Layer 1: 战略意图层 (0.1~0.3Hz)                      │
│  载体: AkiClaw-Node (云端 LLM+VLM)                    │
│  问题: 我在哪？场景是什么？总目标是什么？               │
│  输出: 任务目标 + 约束条件 → 下发到Layer 2             │
├──────────────────────────────────────────────────────┤
│  Layer 2: 战术决策层 (1~5Hz)                          │
│  载体: 地面站本地PC（小模型 + 规划算法）               │
│  问题: 可行路径？最优路线？当前模式(激进/保守/抗风)？  │
│  输出: 路径指令 + PID模式选择 → 下发/上传             │
├──────────────────────────────────────────────────────┤
│  Layer 3a: 行为执行/感知层 (30~100Hz)                 │
│  载体: 机载树莓派 (YOLO + 目标检测)                   │
│  Layer 3b: 空间定位层 (10~50Hz)                       │
│  载体: 机载协处理器 (SLAM / 视觉里程计)               │
│  问题: 目标在哪？障碍？精确位置？                      │
│  输出: 控制修正量 → 下发到Layer 4                     │
├──────────────────────────────────────────────────────┤
│  Layer 4: 实时控制执行层 (200~2000Hz)                 │
│  载体: STM32 + AkiClaw-Mini                           │
│  职责: 保命/PID/动力学/执行链（见1_1_可落地自动PID）  │
│  接口: AkiEnergy + 四档参数 + 振荡能量监控            │
└──────────────────────────────────────────────────────┘
```

## 二、Layer间通信协议

### 2.1 Layer 1 → Layer 2（任务指令，JSON，0.1~0.3Hz）

```json
{
  "mission_id": "m_20250801_001",
  "task_type": "point_to_point",
  "target": {"x": 12.5, "y": -3.2, "z": 2.0, "frame": "NED"},
  "constraints": {
    "max_speed_mps": 3.0,
    "avoid_zones": [{"center": [5.0, 0.0], "radius": 2.0}],
    "priority": "efficiency"   // or "safety" / "stealth"
  },
  "scene_summary": "Open field, light wind from NW, no moving obstacles detected",
  "expires_at_s": 60
}
```

### 2.2 Layer 2 → Layer 3/4（模式指令，二进制高效包，1~5Hz）

```c
typedef struct __attribute__((packed)) {
    uint8_t  mode;          // 0=Hover, 1=Navigate, 2=Track, 3=Emergency
    uint8_t  pid_profile;   // 0=Conservative, 1=Normal, 2=Aggressive
    float    target_pos[3]; // NED目标位置 (m)
    float    target_vel[3]; // 期望速度 (m/s)
    uint8_t  risk_budget;   // 允许的最大风险等级 (0~3)
    uint16_t checksum;
} AkiTacticalCmd;           // 每200ms发送一次(5Hz上限)
```

### 2.3 Layer 4 → Layer 3/2（状态上报，二进制，50Hz）

使用`AkiFeaturePacket`（见1_1版四层架构），包含`E_osc/kurtosis/risk_level`等。

Layer 2通过状态上报更新PID模式选择，形成完整闭环。

## 三、故障降级（天然多级）

```
正常工作: L1→L2→L3→L4（全链路）
         ↓
断云端（L1失联）:
  L2 用最后已知任务目标继续自主规划
  → L3/L4照常运行，无感知变化
         ↓
断视觉（L3失效）:
  L4 切换纯惯性自稳模式
  L2 切换预定航点导航模式（不依赖实时视觉）
         ↓
断通信（L2+L3均失联）:
  L4 安全模式：悬停 → 缓慢降落 → AirMode关闭
         ↓
L4崩溃:
  硬件看门狗触发 → 全电机安全关断(关断想坠机吗？重启啊！)
```

**关键设计原则**：  
> L4永远不依赖上层。任何上层信号超时（>500ms未收到），L4自动进入安全悬停。  
> 意图越高层，断开后越优雅降级（不是直接失控）。

## 四、AkiRouter关键优势（相对现有方案）

### 4.1 意图分层，不是指令分层

传统多层飞控：每层传具体控制指令（速度/角度）。  
AkiRouter：**上层传意图+约束，下层反馈状态**，两者双向路由。

- 云端只说"去那里，优先安全"，不说"向右飞，速度2m/s"
- Layer 4只说"已振荡，风险等级2"，不说"需要调低TPA到0.4"

**每层只知道自己该知道的，各层独立可替换。**

### 4.2 双向：意图下行 + 状态上行

```
L1战略意图 ──────→ 约束/目标
                   ↑
L2战术决策 ←───── 机器人能力/当前状态
         ──────→ 路径/模式
                   ↑
L3感知响应 ←───── 精确位置/障碍
         ──────→ 控制修正量
                   ↑
L4实时执行 ←───── PID参数/模式
         ──────→ E_osc/kurtosis/健康状态
```

### 4.3 天然泛用

换机器人形态 → 只改L4的执行器配置和AkiEnergy参数  
换任务场景 → 只改L1/L2的任务逻辑  
L3(SLAM/YOLO)和L4(PID动力学)完全复用

## 五、各层技术选型（更新+具体）

### Layer 1（云端，0.1~0.3Hz）

| 组件 | 推荐 | 说明 |
|------|------|------|
| LLM | GPT-4o / Claude / Gemini | 场景语义理解、任务规划 |
| VLM | GPT-4o Vision / LLaVA | 视觉场景理解 |
| 框架 | AkiClaw-Node（未来实现）| 统一封装，支持多模型切换 |
| 通信 | HTTPS / WebSocket | 云→地面站 |

**注意**：L1每次推理延迟可达1~5s，完全允许——战略层本来就是低频。

### Layer 2（地面站PC，1~5Hz）

| 组件 | 推荐 | 说明 |
|------|------|------|
| 路径规划 | A* / RRT* | 已知地图；动态障碍加层 |
| 小模型（可选）| Phi-3-mini / Llama 3.1 8B | 本地跑，战术推理 |
| 状态管理 | 简单有限状态机（FSM）| 大多数情况足够 |
| 通信 | MAVLink over WiFi/4G | 标准协议兼容Ardupilot |

### Layer 3a（机载树莓派，30~100Hz）

| 组件 | 推荐 | 说明 |
|------|------|------|
| 目标检测 | YOLOv8n / YOLOv9t | 树莓派5可跑（~20FPS）|
| 视觉运算 | OpenCV | 检测后处理、坐标变换 |
| 与L4通信 | UART（MAVLink或自定义）| 低延迟 |

### Layer 3b（机载协处理器，10~50Hz）

| 组件 | 推荐 | 说明 |
|------|------|------|
| SLAM | ORB-SLAM3 / OpenVINS | 视觉+IMU融合 |
| 硬件 | 树莓派+深度摄像头 / Jetson Nano | 按性能需求选 |
| 建图输出 | 局部点云或稀疏地图 → L2 | 实时性优先 |

### Layer 4（STM32，200~2000Hz）

见`1_1_可落地自动PID系统.md`完整描述。  
关键接口：接受`AkiTacticalCmd`，输出`AkiFeaturePacket`。

## 六、AkiEmotionModule：旁路高速路由（情绪辅助决策）

**位置**：夹在L2和L3之间的旁路，不替代正常规划，提供高速偏向信号。

```python
class AkiEmotionModule:
    """
    把多维状态快速压缩成 [-1, +1] 情绪信号
    负值 = 不舒服（需要收参/降速）
    正值 = 舒适（可以激进）
    """
    def evaluate(self, state) -> float:
        discomfort = (state.E_osc > 0.7*DANGER) + \
                     (state.energy_waste > 2.0) + \
                     (state.kurtosis > 5.0) + \
                     (state.torque_adequacy < 0.5)
        comfort    = (state.E_osc_trend < 0) + \
                     (state.tracking_error < GOOD_THRESH) + \
                     (state.energy_waste < 1.3)
        return (comfort - discomfort) / 4.0
```

情绪信号对各层的影响（旁路，不走正常指令通道）：
- L4：直接触发四档切换（最快响应）
- L3：调整安全距离余量
- L2：建议调整策略（conservative/aggressive模式偏向）
