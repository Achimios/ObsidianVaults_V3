## 一、白噪音 vs Perlin 噪音：特征 & 区别

### 1. 白噪音（White Noise）

- **时域**：完全随机、无关联、前后无关、跳变剧烈
- **频域**：**全频率幅度几乎相等**（平坦谱）
- **听感 / 视觉**：沙沙粗噪、雪花点、无任何平滑趋势
- **特点**：无记忆、不可预测、能量均匀

### 2. Perlin 噪音（梯度噪声）

- **时域**：**连续、平滑、可导、相邻点强相关**
- **频域**：**天然低频多、高频少**（近似 1/f 谱）
- **听感 / 视觉**：渐变、流动、自然、无突兀跳变
- **特点**：可控平滑度、适合 “自然变化”

### 最简区别一句话

- 白噪：**全频乱跳**
- Perlin：**平滑渐变、低频为主**

## 二、公式 / 实现思路（极简可落地）

### 1. 白噪音

最直接：

plaintext

```
sample = 2 * rand() - 1  // [-1, 1] 均匀随机
```

无任何频率偏好，完全平坦。

### 2. Perlin 噪音（1D 最简思路）

1. 建网格点，每个点分配随机**梯度向量**
2. 计算当前点到网格点的偏移
3. 做**点积 + 平滑插值**（fade: 6t⁵−15t⁴+10t³）
4. 输出一个平滑连续值

数学形式（1D 示意）：

n(x)=lerp(dot(g0​,x−x0),dot(g1​,x−x1),fade(x))

实际代码直接套经典 Perlin 1D 实现即可。

## 三、倍频程（Octave）对 Perlin 的影响

Perlin 一般不用单层，而是 **fBm（分形布朗运动）**：

- **octave**：叠加多少层谐波
- **lacunarity**：每层频率倍数（常用 2）
- **gain**：每层幅度衰减（常用 0.5）

效果：

- octave **少**：很平滑、模糊、几乎无高频
- octave **多**：细节变多、变 “碎”、高频变丰富
- gain **越小**：高频衰减越快，整体越柔和

简单记：

**octave 控制细节多少，gain 控制高频有多弱**

## 四、能否控制具体频率？（30Hz 强、200Hz 弱）

### 直接结论：

- **白噪音**：本身**不挑频率**，能量平坦
- **Perlin**：只有**整体 1/f 趋势**，**无法直接指定 30Hz / 200Hz 这种精确频率幅度**

### 你要的效果怎么做？

正确路线：

1. 生成噪声（白 / Perlin）
2. **加滤波器 / EQ**
    
    - 30Hz 附近：增益提升
    - 200Hz 以上：低通衰减
    
3. 再做总增益

不是噪声原生参数可调，是 **噪声 + 滤波 + 增益** 组合。

---

## 五、还有哪些噪声？分别适合什么场景？

### 1. 粉红噪声（1/f）

- 频谱：能量与频率成反比
- 场景：音箱测试、自然底噪、背景音乐基底、均衡参考

### 2. 布朗 / 红噪声（1/f²）

- 极低频极强、高频极弱
- 场景：缓慢漂移、温度漂移、海浪、大尺度运动、电压漂移

### 3. Simplex 噪声

- Perlin 改良，无方块感、计算更快
- 场景：高质量平滑噪声、3D 纹理、运动轨迹

### 4. Value 噪声

- 简单、块感明显、性能高
- 场景：低配设备、简单扰动、低精度模拟

### 5. 脉冲 / 尖峰噪声

- 随机突发尖峰
- 场景：数字干扰、故障音、毛刺、信号损坏模拟

### 6. 混沌噪声（洛伦兹、逻辑映射）

- 确定性混沌、可控混沌度
- 场景：混沌系统、非周期复杂信号、特殊模拟

## 二、机器人噪声 / 干扰 实用优先级（从高到低）

按：**出现概率 > 影响严重性 > 仿真必要性** 排序

### 1. 电学噪声（所有机器人必存在，最高优先级）

1. **热噪声 / 白噪**
    
    - 来源：电阻、ADC、传感器前端
    - 表现：宽带平坦小幅度噪声
    - 模型：高斯白噪
    - 用途：传感器仿真、ADC 精度测试
    
2. **1/f 闪烁噪声**
    
    - 来源：MOS 管、电机、传感器低频漂移
    - 表现：低频强、高频弱
    - 模型：粉红噪声
    - 用途：长时间漂移、稳定性测试
    
3. **电源纹波**
    
    - 来源：开关电源、BEC、线性稳压
    - 表现：50Hz/100Hz / 开关频率谐波
    - 模型：正弦多峰 + 窄带干扰
    - 用途：电源抗扰测试
    
4. **EMI 传导 / 辐射噪声**
    
    - 来源：电机、电调、舵机、开关动作
    - 表现：尖峰脉冲 + 宽带噪声
    - 模型：脉冲噪声 + 白噪
    - 用途：EMC 仿真、硬件抗干扰
    
5. **量化噪声**
    
    - 来源：ADC、编码器、采样
    - 表现：阶梯状小噪声
    - 模型：均匀白噪
    - 用途：控制精度、分辨率分析
    

### 2. 动力学噪声（机械 / 运动，次高优先级）

1. **电机转矩纹波**
    
    - 表现：速度 / 电流周期性波动
    - 模型：基频正弦 + 谐波 + 小噪声

## 一、先把你系统的「死穴」说清楚

你这套系统：

#### 核心矛盾

- 想**延迟最低、响应最快**

    - 相位裕度 < 20°：**快速发散**
    - 共振 + 相移 → 正反馈 → 瞬间失控
    

#### 结论（非常重要）

**不能用固定 PID + 固定滤波**

必须：

1. 平时：**极限激进、最低延迟**
2. 一旦检测到【快要发散】：
    
    **毫秒级联动：PID 衰减 + 滤波收紧 + 陷波打开 + 延迟补偿**
3. 稳住后：**平滑恢复激进参数**

## 二、毫秒级：怎么判断「快要发散 / 已经危险」

只用**轻量、2kHz 无压力**的特征，

滑动窗 **8~32 点（4~16ms）**，完全满足你毫秒～秒级需求。

### 2.1 最靠谱 4 个特征（优先级从高到低）

#### 1）峰度（Kurtosis）突增 ★★★★★

- 正常平稳：≈3
- 轻微振荡：4~6
- **快发散：>7**
- 最适合：共振、振荡、冲击、洗桨、机体抖动
- 计算：滑动 16 点，代价极低

#### 2）Teager 能量算子（TEO）★★★★★

- 专门抓：**快速变化、振荡、能量爆发**
- 比看幅值灵敏 5~10 倍，**几乎 0 延迟**
- 公式（只用 3 个点）
    
    Ψ[x(n)]=x(n)2−x[n−1]x[n+1]
- 触发：连续 3~5 帧超阈值

#### 3）D 项爆冲 + PID 输出饱和 ★★★★

- D 对振动 / 共振最敏感
- 现象：
    
    - D 项剧烈抖
    - PID 输出频繁撞上下限
    
- 这是**发散前最典型电信号**

#### 4）简易滑动谱：共振峰能量 ★★★★

- 只盯你机体 / 电机的**共振频率 fr**（比如 60~150Hz）
- 该频点能量一爆 → 共振被激发 → **必发散**
- 2kHz 下只算 1~3 个频点，几乎不占 CPU

### 2.2 直接可用的分级判定

- **Level 0 正常**：平稳，随便飞
- **Level 1 预警**：峰度 4~7 或 TEO 略超
- **Level 2 危险**：峰度 >7 + D 项狂抖
- **Level 3 紧急**：共振峰爆 + PID 输出饱和

## 四、滤波相移、群延迟、通信延迟怎么解决

### 4.1 群延迟 & 相移补偿（最简单有效）

ecomp​=e+k⋅Δte−eprev​​⋅τ

- 不要等当前采样，用 “上一帧 + 斜率” 预估现在

### 4.3 D 项必守规则（90% 人炸在这里）

- D 项低通截止 ＜ 主环低通截止
- D 项最大输出 ＜ P 项
- 振荡 / 发散时：**优先砍 D，不是只砍 P**

## 五、PID 极限拉高又不炸的关键规则

#### 1）P 只拉到「临界振荡的 80%」

1. 先关 D，慢慢提 P，直到**刚要抖**
2. 然后 P × 0.8 作为上限
    
    这是 “最快又不炸” 的物理上限。

#### 2）D 只负责阻尼，不负责快

- D 大 → 放大振动 → 激发共振 → 发散
- D 正确用途：**抑制超调，不提速**

#### 3）积分要 “瘦”

- I 大 = 延迟大 = 相位裕度低 = 易发散
- 只给刚好消除静差的最小值

#### 4）必须开抗积分饱和

发散时积分一炸，任何 PID 都救不回来。

规则：

- 输出在限幅内才积分
- 超幅就**停止积分 + 缓慢回退**

#### 5）共振用「陷波 Notch」，别只用低通

- 低通滤共振：延迟巨大，响应废掉
- 陷波：**只砍共振频点，几乎不影响整体延迟**
    
    这是你能**高 PID + 抗共振**的核心。

## 一、物理上为什么成立？（一句话讲透）

你机体、电机里的**无效能耗**，主要就三块：

1. 电机电流震荡、来回正反换向 → 铜损、铁损 ↑
2. 机体共振、结构振动 → 机械摩擦、发热、声能耗散
3. PID 超调 / 振荡 → 控制量白抖，电就白耗

这些东西有一个共同特点：

**都表现为「快速振荡、突变、来回波动」**

而：

- 能量算子 / 振荡积分
    
    **算的就是：单位时间内，信号 “抖、震、冲” 的总能量**

这部分能量 ≈ **90% 都是无效内耗、浪费掉的能耗**

你不需要算出绝对焦耳（那需要电机模型），

你只需要**相对值**：

- 哪个 PID 震荡能量小 → 内耗小、省电、更稳
- 震荡能量突然飙升 → 内耗暴涨 + 快要发散

## 二、你真正能用的 4 种算法（按实用度排序）

全部满足：

- 2kHz 实时
- 100ms ~ 数秒滑动窗口
- 可和功率计校准
- 可预测、可对比 PID

### 1）滑动窗口 **震荡能量积分（最核心、首选）**

适合：**100ms～数秒 内耗统计、PID 对比、预测**

#### 定义（工程简化版，直接用）

对角速度 / 加速度 / 电机 RPM / PID 输出 做：

Eosc​(t)=N1​∑k=t−N+1t​(y[k]−yˉ​)2

- y：姿态角、角速度、RPM、电流、PID 输出
- yˉ​：窗口内均值（去掉趋势，只留震荡）
- N 对应窗口：
    
    - 100ms @2kHz：N=200
    - 500ms：N=1000
    - 1s：N=2000
    - 数秒：分段累积平均
    

#### 物理意义

- 数值越大 = 振动 / 震荡越剧烈 = **内耗越大、越费电**
- 平稳飞行 / 运动：Eosc​ 很小
- 共振、PID 发散、洗桨：Eosc​ 瞬间暴涨

#### 优点

- 计算极简单：平方、求和、平均
- 延迟极低，**真正毫秒级**
- 和实测功率**高度线性相关**

---

### 2）Teager 能量算子累积（TEO）

适合：**瞬态冲击、洗桨、微秒～毫秒级高能浪费、发散前兆**

离散公式：

Ψ(y[n])=y[n]2−y[n−1]y[n+1]

短时累积（10~100ms）：

Eteager​=∑∣Ψ(y[n])∣

#### 用途

- 抓**快速冲击、共振激发、瞬间高能耗**
- 比单纯平方更敏感：**小幅度但变化快的振动，也能抓到**
- 非常适合：电机突震、洗桨、撞击、EMI 高能毛刺

---

### 3）PID 输出波动能量（直接对应电耗）

适合：**对比高低 PID、控制量本身浪费**

Epid​=N1​∑(u[n]−uˉ)2

- u[n]：PID 输出（油门 / 力矩 / 电机占空比）
- 意义：
    
    - PID 来回抖 → 电机频繁加减速 → 耗电发热
    - 同样动作，Epid​ 小 = 控制平滑、省电
    

---

### 4）振荡幅值平方积分 ISA（工控标准）

适合：**长时间、全时域总浪费统计**

ISA=∫∣y(t)−ytarget​∣2dt

- 可以分段：1s、5s、全程
- 直接得出：**这段动作里，震荡浪费总分**
- 用来给 PID 排名：分数越低 → 能效越高、越稳

## 三、怎么和「实际功率监测」结合？（关键）

你有功率 / 电流采样，这套算法立刻从 “相对值” 变成**准绝对功耗预测**。

### 步骤（非常简单）

1. **做一次校准**
    
    - 飞机 / 车 / 机械臂静置：
        
        - 记录：功率 P0​（静态损耗）
        - 记录：震荡能量 E0​≈0
        
    
2. **运动时**
    
    Pwaste​≈Pmeas​−Pbase​−Pload​
    
    - Pmeas​：实测功率
    - Pbase​：基础功耗
    - Pload​：有用功（爬升、负载、速度）
    
3. **用震荡能量拟合**
    
    Pwaste​≈k⋅Eosc​+b
    
    - 在线跑一次最小二乘，得到系数 k、b
    - 以后：**只看 E_osc，就能实时推算浪费功率**
    

### 你能得到

- 当前：**震荡浪费功率 W**
- 100ms：**累计浪费能量 J**
- 全程：**总震荡耗电 Wh**

## 四、高低 PID「同动作能耗对比」怎么搞？

必须**控制变量**，否则对比无意义。给你工程标准方案：

### 1）固定动作基准

- 飞同样航线 / 机械臂走同样轨迹 / 车走同样路径
- 风速 / 负载尽量一致
- 分别跑：

### 2）统一计算 3 个指标

2. **PID 输出波动能量 Epid​**
3. **实测平均功率 Pavg​**

### 3）得出结论

- 高 PID：

- 低 PID：
    
    - Eosc​ 小 → 省电、平顺
    - 但响应慢
    

### 4）给出一个「能效 - 响应综合分」

Score=α⋅Eosc​+β⋅Epid​响应速度指标​

- 分数越高 = 又快又省电又稳
- 可以用来**自动选最优 PID 增益**

## 五、能不能「预测未来几百 ms 内耗 & 发散」？

## 九、最简可执行框架（伪代码 C 风格）

c

运行

```
// 窗口配置
#define WINDOW_100MS  200
float window[WINDOW_100MS];
int idx = 0;

// 每帧 2kHz 执行
void calc_osc_energy(float y, float P_meas)
{
    // 1. 滑动窗更新
    window[idx++] = y;
    if (idx >= WINDOW_100MS) idx = 0;

    // 2. 均值 & 震荡能量
    float mean = sum(window) / WINDOW_100MS;
    float E_osc = sum_sq_diff(window, mean) / WINDOW_100MS;

    // 3. Teager短时能量
    float teager = y*y - y_prev*y_next;

    // 4. 预测未来100ms
    float dE_dt = (E_osc - E_osc_prev) / dt;
    float E_pred = E_osc + dE_dt * 0.1f;

    // 5. 内耗预警 & PID决策
    if (E_pred > E_THRESH || dE_dt > SLOPE_THRESH) {
        // 内耗即将暴涨 + 快发散
        reduce_PID_gain();
        tighten_filter();
        enable_notch();
    }

    // 6. 与功率结合
    P_waste = k * E_osc + b;
}
```
