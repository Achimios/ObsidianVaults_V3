## 五阶段单人开发路线（稳·狠·准）

### Stage 1：AkiLabs内核（3~6个月）🎯 当前阶段

- 统一刚性/柔性在线辨识（已有框架）
- 自适应控制库（MCU实时，已有四档）
- 振荡能量/TEO/峰度模块（已有C代码框架）
- 先验注入v1：桨叶参数→增益起点
- 首批机型：**穿越机 + 单轮自平衡 + 轮式差速**
- 开源，打出名气（v0.1~v0.3节点）

**产出物清单：**
- [ ] AkiEnergy C库（振荡能量 + TEO + 峰度，BSD/MIT协议）
- [ ] 动态PID四档切换 + 平滑过渡（穿越机实测数据附录）
- [ ] Dshot-RPM失配检测 + 动力根因分类
- [ ] README第一行宣言（见下方）

### Stage 2：AkiKinematic 1.0

- 基础FK/IK引擎（冗余自由度支持）
- 机械臂 / 云台适配
- 轨迹生成（最小加加速度/样条）

### Stage 3：多足动力学

- 多体动力学模型（低阶模态近似）
- 质心控制、步态库（Trot/Walk/Bound）
- 四足实机验证（对标ANYmal思路但更轻量）

### Stage 4：人形双足

- ZMP稳定性分析
- 躯干柔性补偿
- 奔跑控制（对标UZH双足平台）

### Stage 5：全平台统一

- 一份配置文件切换所有机器人形态
- 公开benchmark（对比BetaFlight / ArduPilot / FAST Lab）
- 学术论文（如果需要）：《泛用机器人振荡能效最优控制》

## AkiKinematic vs AkiLabs 分工

```
AkiKinematic  →  "去哪里" — 期望轨迹生成（FK/IK/步态/ZMP）
      ↓
AkiLabs       →  "怎么去" — 实机执行（稳/快/省电/不炸）
```

两者完全解耦。AkiKinematic输出平滑期望轨迹，AkiLabs做真实物理世界的自适应执行。可以独立做研究，也可以无缝协作。
