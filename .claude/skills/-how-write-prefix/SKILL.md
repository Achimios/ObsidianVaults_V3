# V3 命名前缀规范

## 前缀速查表

| 前缀 | 含义 | 示例 |
|---|---|---|
| `-` | Vault 级，人类+AI 均可读；架构相关 | `-Vault空间/`, `-CONTEXT.md`, `-PRD.md` |
| `_` | 临时类，通常人类不需要主动读 | `_TEMP/`, `_ScreenShots/` |
| `__` | 完全不用管，通常 archived | `__trash/` |
| `@` | 代码相关 | `@Code_Project/`, `@python.instructions.md` |
| `z_` | 仍在讨论/待更新的文件夹 | `z_范数空间/`, `z_文本格式的讨论/` |
| `*.0_` | 对话级原始内容，待提炼 | `0.0_Doubao_范数空间.md` |
| `*.1_` | 一轮提炼后；`*.2_` 为再次提炼 | `0.1_Copilot_范数空间.md` |
| `*-0_` | 仍在更新的方案性内容 | `1-0_可落地自动PID系统` |
| `*-1_` | 方案经第一次更新后 | `1-1_可落地自动PID系统` |
| `0_` | 给人看的，AI框架配置/架构相关 | `0_AI接入口在哪？.md`, `0_程序结构.md` |
| `1_`, `2_` | 给人看的局部接入点 | `1_人类接入手册.md`, `1_数学原理.md`, `2_可落地意图控制层.md` |
| 无前缀或自定义 | 普通笔记/代理文件，按场景决定 | `🎮明日香.agent.md` |

## 优先级规则

- 前缀冲突时：`-` > `@` > `z_` > 无前缀
- 文件夹和文件使用相同前缀体系
- `0_`, `1_` 与 `*.0_`, `*.1_` 不同：前者是人类接入点，后者是对话提炼阶段

## 示例对照

```
-人类思维笔记/
  z_范数空间/              ← 仍在讨论的文件夹
    0.0_Doubao_范数.md     ← 原始对话
    0.1_Copilot_范数.md    ← 一轮提炼
  -Base演示_希尔伯特/      ← 稳定示例，人类+AI均读
    -subWikis/
      内积空间.md
```
