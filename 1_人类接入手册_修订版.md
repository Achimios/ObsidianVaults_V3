## 二、💬 每次对话怎么开始

1. 看 [[-where_continue]]，选择最近工作的层
2. 复制对应 `-CONTEXT.md` 路径，粘贴到 AI 对话框
3. 说你要干什么，AI 会读 CONTEXT 恢复状态

> [!tip] VS Code workspace 多根目录时，每个项目的 `copilot-instructions.md` **都会被注入**，注意叠加效应。 构建新 Vault 版本、迁移旧版本时可以多根，新旧版本指令临时共存。

## 四、🎭 AI 指令层（手动触发）

### 4-1 · 附加指令（手动Attach，叠加注入）

在当前对话上下文里**叠加**额外规则，不替换现有上下文。

- 任意位置的任意文件（比如 未填写`applyTo`的`.instructions.md`，或 `带我征服宇宙.xml`）
- 在 Chat 输入框手动 attach 或粘贴 文件路径

### 4-2 · Prompt 文件（手动 `/` ，叠加注入）

- 位置（项目级）：`.github/prompts/`
- 位置（用户级）：`%USERPROFILE%\AppData\Roaming\Code\User\prompts`
- 格式：**`*.prompt.md`**
- 触发方式：Chat 输入框输入 `/文件名` 斜杠命令
- 支持变量：`${workspaceFolder}`、`#file:路径` 等动态引用

### 4-3 · Agent（叠加 or 独立上下文，取决于召唤方式）

- 位置（项目级）：`.github/agents/`，格式：`*.agent.md`（`*.md` 也识别，==bug==）
- 位置（用户级）：`%USERPROFILE%\AppData\Roaming\Code\User\prompts`（理论应丢 `\agents`，==bug==）
- 召唤为**叠加**：Chat 窗口左下角 Agent 图标列表，或对话让主代理自己触发
- 召唤为**子代理（==独立上下文==）**：通过 `runSubagent` 工具，或对话让主代理自己触发

**Claude Code 中召唤**：用 `/subagent` 或同样用对话触发。

> [!NOTE] 实测结论（2026-03-30）
> 子代理可以成功召唤，上下文干净。⚠️ 但 `copilot-instructions.md` / `AGENTS.md` **仍会被注入**，需 Sonnet 级别模型，Haiku 很可能压根不召唤。非真正沙箱隔离。详见 [[软件操作手册/VSCode-Copilot指令机制参考]]。

> [!tip] Chat 窗口右上角齿轮图标，可查看当前已加载的 Agent、Skills 等配置。

## 五、⚙️ 必须配置的 settings.json

> [!tip] Chat 窗口右上角三个点 → Chat Settings，搜索框输入关键词（如 `useClaude`）可快速定位。

``` json
{
  "chat.agent.maxRequests": 300, // 长任务不中断（搭配 PARDON 弹窗机制）

  "chat.useClaudeMdFile": false, // Copilot不读取CLAUDE.md

  "chat.useNestedAgentsMdFiles": true, // 启用子目录 AGENTS.md（项目局部指令）
  
  "chat.allowInvocationsFromSubagents": true // 允许子代理再召唤子代理
  
  Editor自动保存 记得打开，设置为 delay 或者 失焦
}
```

## 七、✂️ Prompt 分隔符速查

最常用：直接用 XML 标签把指令和数据分开：

```xml
<task>把下面的邮件改写成正式商务风格</task>
<email>Hi team, lmk ASAP!</email>
```

| 用途 | 分隔符 |
|---|---|
| 嵌套引用、防注入（Claude 天然偏好） | `<tag>...</tag>` XML 标签 |
| 代码 / 逐字保留文本 | ` ``` ` 三反引号 |
| 长 prompt 结构化分节 | `### 标题` Markdown 标题 |
| 轻量行内引用 | `> 内容` 引用块 |

> ⚠️ Obsidian 笔记里 `<tag>` 会被隐藏渲染；**发给 AI 的文本**直接裸写 XML 完全正常。

详细用法、防注入、多层嵌套 → [[AI提示词分隔符指南]]

## 七、🔑 注入验证（Magic 校验机制）

所有注入文件（`copilot-instructions.md`、`AGENTS.md`、`*.instructions.md`、`.agent.md` 等）的**顶部**（frontmatter 下方）均附有各自独立的验证码。

**用途**：任何框架（Copilot / Claude Code / OpenClaw / Codex）出现异常行为时，直接开口问：

> 「目前启用了哪些 magic？」

AI 如能说出正确的那句（每个文件不同），说明该文件已成功注入。答不出或答错，说明对应文件没有被加载。一句话秒判注入状态。

所有 magic 的完整目录在根目录 `AGENTS.md` 的「注入验证机制」章节。
